## 摘要
随着推理模型生成数万token长的思维链，KV缓存日益成为部署瓶颈。现有的缓存驱逐方法依据注意力权重对token排序，但在长推理轨迹中，注意力权重是噪声较大的重要性代理，且该方法迫使模型物化注意力矩阵，从而禁止在生产推理中使用融合内核。在本工作中，我们转而使用一种称为顿悟分数（epiphany score）的度量对token评分：该分数从前向传播中直接读取模型内部表示的变化，无需注意力矩阵且额外状态可忽略不计。由此产生的缓存驱逐方法EpiKV无需训练、分类器或定制内核，可直接在FlashAttention推理栈中无需修改地使用——其可行上下文长度比基于注意力评分的缓存驱逐方法扩展16倍。EpiKV通过剔除趋势和基于token位置的归一化作用于顿悟分数：我们在数学推理链上验证了中间层对顿悟分数的贡献（低层影响中性，中上层消极），并通过因果滚动z分数消除位置趋势。在4096 token缓存下，EpiKV在MATH-500上达到72%，与最强的基于注意力的基线持平（ThinKV 71%，H2O 67%）；在8192 token下，其滞后归一化KV变体在AIME-2024上达到37%，优于最优基线（33%），速度提升高达2.8倍。

## Abstract
As reasoning models emit chains of thought tens of thousands of tokens long, KV cache increasingly becomes a deployment bottleneck. Existing cache eviction methods rank tokens by attention weight, which is a noisy importance proxy in long reasoning traces, and prohibits the use of fused kernels in production inference by forcing the model to materialize the attention matrix. In this work, we instead score tokens with a metric we term the epiphany score: the change in the model's internal representation, read directly from the forward pass with no attention matrix and negligible extra state. Our resulting cache eviction method, EpiKV, requires no training, classifier, or custom kernel, and can be used directly in FlashAttention inference stacks unchanged -- scaling to a 16x longer feasible context than attention-based scoring. upper-mid layers negatively) and remove a positional trend with a causal rolling z-score. - **整体含义**：提出一种无需注意力矩阵的缓存淘汰方法EpiKV，使用模型内部表示变化（“顿悟分数”）直接评分，兼容FlashAttention，在精度持平甚至更优的同时，显著提升速度并支持更长上下文。

## 2. 论文提出的方法论：核心思想、关键技术细节

### 核心思想
- **顿悟分数（epiphany score）**：token的重要性由其生成时引起的残差流状态变化幅度决定。变化大的token（如中间结果、推理转折点）被认为是“顿悟”token，更值得保留。
- 不需要注意力矩阵：仅使用前向传播中自然暴露的隐藏状态（hidden states）和KV缓存中的键值向量。

### 关键技术细节
1. **两层带解剖**：
   - 在32层推理模型上，通过反事实遮挡实验发现两层带：
     - **Band A（层7-13）**：隐藏状态L2变化（gl(t) = ∥hl(t)-hl(t-1)∥₂）与token重要性呈正相关（Spearman ρ>0）。
     - **Band B（层18-25）**：与重要性呈负相关。
   - 结合为正负信号，得分 s(t) = mean(gl Band A) - mean(gl Band B)。

2. **时间趋势校正**：
   - 原始信号存在位置相关趋势（早期token变化大但未必重要，后期token变化小但关键）。
   - 使用因果滚动z-score：zl(t) = (gl(t) - μl(t)) / (σl(t) + ε)，其中μl、σl来自前缀窗口（w=64）。然后s(t) = z10 - z21（取两带代表层）。

3. **多策略族**：
   - **隐藏状态族**：HS-variance (原始均值差)、EpiKV (z-score校正)、Band-adaptive。
   - **KV向量族**：KV-key/val variance、Lag-KV（借鉴LagKV的滞后归一化但改为因果版本）。
   - **混合族**：Attn×HS（积分注意力与隐藏状态结合，但需eager kernel）。

4. **兼容性**：定义“FA2兼容”策略——仅使用已缓存的键值和隐藏状态，不请求注意力矩阵，可在FlashAttention中运行。
