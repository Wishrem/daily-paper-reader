## 摘要
自注意力是Transformer性能的核心，并且在长上下文长度下通常是Transformer中最昂贵的部分，因为其成对令牌交互的计算量随序列长度呈二次增长。标准的密集注意力对每个令牌应用相同的注意力头集合，无论令牌难度或信息内容如何。这种均匀激活会浪费计算资源，尤其是当序列变长且注意力成本迅速增加时。我们提出分组查询专家（GQE），这是一种在分组查询注意力（GQA）之上的混合专家层。在每个GQA组中，路由器为每个令牌选择k个查询专家头，而所有键值（KV）头保持密集且不变。因此，GQE保留了GQA的KV缓存优势，仅减少了活跃的查询头计算。在250M参数规模下，使用固定的30B令牌预算，GQE在下游准确率上与全活跃的GQA基线相当，同时每个令牌仅激活一半的查询头。

## Abstract
Self-attention is central to Transformer performance and is often the most expensive part of the Transformer at long context lengths because its pairwise token interactions scale quadratically with sequence length. Standard dense attention also applies the same set of attention heads to every token regardless of token difficulty or information content. This uniform activation can waste compute, especially as sequences grow longer and attention cost increases rapidly. We propose Grouped Query Experts (GQE), a mixture-of-experts layer on top of grouped-query attention (GQA). Within each GQA group, a router selects k query-head experts per token while all key-value (KV) heads remain dense and unchanged. Thus, GQE keeps the KV cache benefits of GQA and reduces only the active query-head computation. On a fixed 30B token budget at the 250M parameter scale, GQE matches the all-active GQA baseline in downstream accuracy while activating half the query heads per token.

## 论文总结：分组查询专家 (Grouped Query Experts: GQE)

### 1. 核心问题与整体含义（研究动机和背景）

- **核心问题**：Transformer模型的自注意力机制在长序列场景下计算成本极高（与序列长度呈二次方增长），且其**均匀激活**所有注意力头的方式存在效率浪费。对于内容性不足的令牌（如标点、停用词），激活所有头是不必要的。
- **研究动机**：能否将**条件计算**（如混合专家模型）的思想从Transformer的MLP模块**迁移到自注意力模块**中，使得模型能够根据每个令牌的特性，只激活其所需的注意力头？
- **整体含义**：本文旨在设计一种更高效的注意力机制，在**保持模型精度的前提下，显著降低自注意力的计算量**，尤其是在处理长序列时。

### 2. 论文提出的方法论：核心思想与关键技术细节

**核心思想**：在**分组查询注意力 (GQA)** 的基础上，引入**混合专家模型 (MoE)**。将每个GQA组内的多个查询头视为“专家”，通过一个路由器为每个令牌**稀疏地选择**并激活部分查询头，而键值（KV）头保持密集不变。

**关键技术细节**：
1.  **模型结构**：
    -   **基础**：基于**GQA**，将H个查询头划分为G个组，每组共享一个KV头。
    -   **专家池**：在每个GQA组内部，将M个查询头定义为“专家” $E_{g,1}, ..., E_{g,M}$，每个专家拥有自己的查询投影。
    -   **路由器**：对每个令牌 $x_i$ 和每个组 $g$，路由器计算该组内每个专家的分数，并通过 **softmax** 函数将其归一化为概率 $p_{i,g}$。
    -   **稀疏激活**：在每个组内，只选择**top-k个**得分最高的专家进行激活。模型中，k=1，每个组只激活一个专家。
2.  **输出构建**：
    -   **硬拼接**：被选中的专家的输出 $o_{i,g,m}$ 直接拼接起来形成输出的一部分。
    -   **加权求和槽**：为了给路由器提供可微的梯度信号，论文增加一个“再归一化加权和”槽 $\bar{o_i}$。该槽使用所有被选专家的**重新归一化**概率加权求和，形成一个可微分的路径。
    -   **共享头**：增加一个**始终激活**的共享注意力头 $s(x_i)$，提供稳定、与令牌无关的注意力路径，在训练初期稳定模型。
    -   **最终输出**：最终输出由 **硬拼接的专家输出**、**加权求和槽**和**共享头输出**三部分拼接而成，再通过输出投影矩阵 $W_O$ 得到。
3.  **辅助损失**：使用**负载均衡辅助损失**，防止路由器将所有令牌都路由到同一个专家，鼓励专家间的差异化学习。
4.  **计算量分析**：主要计算节省来源于**减少了活跃的查询头**。在本实验设置中（16个查询头/8个KV组，k=1），激活了8个路由专家 + 1个共享头，共计9个查询头，相比全激活的16个查询头减少了约56%的查询计算量。

    -   **评测基准**：HellaSwag、PIQA、ARC-Easy。
-   **对比基线**：
    -   **GQA基线**：标准的分组查询注意力模型，所有查询头保持活跃。

### 7. 优点

-   **创新的迁移思路**：巧妙地将MoE的稀疏计算思想从MLP模块迁移到GQA注意力模块，开辟了优化自注意力计算的新途径。
-   **设计严谨有效**：识别并解决了稀疏路由在注意力模块中可能导致的训练问题（梯度信号弱、训练不稳定），并通过**严谨的消融实验**证明了其提出的加权求和槽和共享头这两个设计元素的必要性。

-   **目标明确**：论文的目标是“在保持质量的同时减少计算”，该目标被清晰地定义并得到了实验验证。
