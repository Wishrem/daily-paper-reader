## 摘要
键值（KV）缓存是一种加速大语言模型推理和生成的有效技术。推理工作负载量大且多样化，这使得有效缓存变得困难。现有的缓存管理策略采用最近最少使用（LRU）策略来驱逐缓存块。然而，LRU会导致多个不相关的工作负载相互清除对方的缓存。为解决此问题，我们集成了自适应缓存，该缓存动态地在最近出现和频繁出现的KV块之间分配缓存空间。评估表明，在合成文档问答工作负载上，与朴素的vLLM相比，该方法将KV缓存命中率提高了最多10.8%，并将首个token生成时间减少了最多12.6%；在真实对话工作负载上，分别提高了2.1%和2.0%。该方法能很好地泛化到批量推理，并在有效适应多样化工作负载的同时展现出清晰的可解释性。

## Abstract
Key-value (KV) caching is a powerful technique for accelerating large language model inference and generation. Inference workloads are large and diverse, which makes them difficult to cache effectively. Existing cache management strategies adopt the least-recently-used policy for evicting cache blocks. However, LRU leads to multiple unrelated workloads flushing each other's caches. To address this, we integrate adaptive caching that dynamically allocates cache space between recently and frequently occurring KV blocks. Evaluations show that it improves the KV cache hit rate by up to 10.8% and reduces time to first token by up to 12.6% over naive vLLM on synthetic document question answering workloads, and 2.1% and 2.0% respectively on real-world conversation workloads. The method generalizes well to batch inference and demonstrates clear interpretability while effectively accommodating diverse workloads.

## 1. 核心问题与整体含义（研究动机和背景）

- **核心问题**：在大语言模型（LLM）推理服务中，KV缓存是一种加速生成的关键技术。然而，现有主流框架（如vLLM）普遍采用最近最少使用（LRU）驱逐策略，在混合工作负载下（如文档问答与多轮对话并存），不同请求之间会相互冲刷缓存，导致命中率低、首个token生成时间（TTFT）增加。
- **整体含义**：LLM推理工作负载具有复杂的访问模式，既存在基于时间局部性的“最近”模式，也存在基于热点文档或活跃用户的“频繁”模式。纯LRU策略无法适应这种混合模式，需要更智能的缓存管理来提升GPU内存利用率和系统吞吐量。

## 2. 论文提出的方法论

- **核心思想**：将经典的自适应替换缓存（ARC）引入LLM推理系统的KV缓存管理，动态平衡“新近度”与“频率”两个维度。
- **关键技术细节**：
  - **双队列结构**：低频率队列（L1 = T1 + B1）存储最近出现过一次的KV块；高频率队列（L2 = T2 + B2）存储出现多次的KV块。命中时从L1提升到L2。
  - **动态分区与幽灵缓存**：物理缓存T1和T2共同占用固定GPU内存；幽灵缓存B1和B2仅存储已驱逐块的元数据（不占KV内存）。当请求命中幽灵缓存时，表明若该队列分配更多空间本该命中，算法据此调整T1和T2的分区比例。
  - **无参数自适应**：算法自动学习工作负载变化，无需人工调整。
- **算法流程**（文字说明）：
    1. 每次请求一个KV块时，检查其在T1、T2、B1、B2中的位置。
    2. 若命中T1或T2：直接返回缓存；若是T1命中，则将该块移动到T2的MRU端。
    3. 若命中B1（幽灵命中）：说明最近频繁出现的块应该获得更多缓存，增加T1大小（p值增大），并从T2驱逐最久未用块以腾出空间。
    4. 若命中B2：增加T2大小（p值减小），并从T1驱逐。
    5. 若完全未命中：根据当前p值选择从T1或T2中驱逐一个块，并将新块放入对应队列。
