## 摘要
上下文密集代理对键值（KV）缓存施加了不同寻常的压力：长前缀在许多短轮次中重复使用，而并发性决定了服务系统能否保持GPU利用率。我们研究了这种情况下4位KV缓存压缩，使用TurboQuant风格的旋转和码本量化作为质量锚点，vLLM FP8 KV缓存作为部署锚点。我们报告了三项贡献。首先，我们将4位KV缓存置于多轮代理工作负载的框架下，其中任务质量、缓存驻留和服务吞吐量必须联合衡量。其次，我们描述了使4位路径稳健所需的实际设计选择，包括非对称K/V处理、Walsh-Hadamard旋转、QJL移除和块尺度变体。第三，我们介绍了在AMD GPU上的服务优化，包括优化的解码注意力内核和UltraQuant——一种使用FP8查询、FP4 KV张量、UE8M0组尺度和CDNA4上原生scaled-MFMA支持的FP4近似路径。在长上下文、多轮代理工作负载中，UltraQuant在缓存压力大的后期轮次中将P50首次令牌时间减少了3.47倍（所有轮次平均减少2.3倍），并将输出吞吐量相比FP8 KV基线提高了1.63倍。

## Abstract
Context-heavy agents place unusual pressure on the key-value (KV) cache: long prefixes are reused across many short turns, while concurrency determines whether the serving system can keep GPUs utilized. We study 4-bit KV-cache compression for this setting, using TurboQuant-style rotation and codebook quantization as a quality anchor and vLLM FP8 KV caching as the deployment anchor. We report three contributions. First, we frame 4-bit KV caching around multi-round agent workloads where task quality, cache residency, and serving throughput must be measured jointly. Second, we describe the practical design choices needed to make the 4-bit path robust, including asymmetric K/V treatment, Walsh-Hadamard rotation, QJL removal, and block-scale variants. Third, we present serving optimizations on AMD GPUs, including optimized decode-attention kernels and UltraQuant, an FP4 approximation path that uses FP8 queries, FP4 KV tensors, UE8M0 group scales, and native scaled-MFMA support on CDNA4. On a long-context, multi-turn agentic workload, UltraQuant cuts P50 time-to-first-token by 3.47x in the cache-pressured late rounds (2.3x across all rounds) and raises output throughput by 1.63x over the FP8 KV baseline.

### 1. 论文的核心问题与整体含义（研究动机和背景）

- **背景**：大型语言模型（LLM）从短文本对话演变为能够执行多轮推理、代码浏览和工具调用的**上下文密集代理**。这类工作负载需要非常长的前缀（系统指令、检索文档、历史计划等）在多轮交互中被重复使用，同时服务系统必须支持高并发以充分利用GPU。
- **核心问题**：随着模型上下文窗口扩展至百万级token，**KV缓存**成为高带宽内存（HBM）的首要消费者。传统的FP8量化（压缩约2倍）虽已硬件原生支持，但在上下文极度密集的场景下仍会因缓存容量不足导致频繁的缓存逐出和重填充，严重拖慢首令牌延迟（TTFT）和整体吞吐量。
- **整体含义**：本文旨在为上下文密集代理场景设计一种**4位KV缓存方案**，在保持可接受任务质量的前提下，最大化缓存容量增益，并通过硬件原生操作避免软件反量化开销，从而在系统层面实现端到端的服务优化。

### 2. 论文提出的方法论：核心思想、关键技术细节

- **核心思想**：采用**TurboQuant风格的旋转+码本量化**作为质量锚点，但针对部署效率进行修改，最终提出**UltraQuant**——一个将反量化折叠进GPU矩阵核心指令的FP4近似路径。
- **关键技术细节**：
  - **非对称K/V处理**：键和值采用不同的量化策略，因为键的错误会扰动softmax分布，而值误差影响较小。
  - **Walsh–Hadamard旋转**：在量化前对键进行正交旋转，将离群能量分散到多个通道，使分布更接近高斯分布，从而有利于标量量化。
  - **去除QJL**：TurboQuant中的QJL组件被移除，因为旋转已经起到了类似的白化效果。
  - **块尺度（Block-Scale）**：替代逐token的ℓ2归一化，每个32通道的块使用自身绝对最大值（absmax）作为尺度，配合一个全局常数`c`将值映射到FP4网格。这使得实现简单且模型无关。
  - **UltraQuant缓存放飞格式**：键和值存储为**FP4 E2M1码 + UE8M0组尺度**（每组32通道，共占用4.25比特/元素）。反量化通过CDNA4的`MFMA_SCALE_F32_*_F8F6F4`指令原生执行，无需软件查找表。
  - **全局常数优化**：离线优化一个常数`c=0.156`，使得FP4网格在旋转后的分布上均方误差（MSE）最小。该常数对所有模型和头部共享。
  - **编码/解码流程**（算法1）：
    - 编码：旋转键 → 分32通道组 → 计算每组的absmax和对应的UE8M0指数 → 对每个坐标四舍五入到最近的FP4码点 → 存储两个FP4码每字节+一个UE8M0字节。
    - 解码：旋转查询并转换为FP8 → 加载FP4码和UE8M0字节 → 通过`code × 2^(scale-127)`反量化 → 利用scaled-MFMA在FP8×FP4路径上直接计算注意力分数。
- **两个实现变体**：
  - **Ultra-TQ**：保留码本量化的优化实现（使用Lloyd-Max质心），通过内核调度、布局优化等提升速度。
  - **UltraQuant**：完全使用FP4硬件原生路径，移除码本，是本文的核心贡献。
