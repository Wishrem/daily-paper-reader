## 摘要
上下文密集型智能体对键值（KV）缓存带来了特殊压力：长前缀在多个短轮次中被复用，而并发性决定了服务系统能否保持GPU利用率。我们针对这一场景研究4位KV缓存压缩，采用TurboQuant风格的旋转和码本量化作为质量锚点，以vLLM FP8 KV缓存作为部署锚点。我们报告三项贡献。首先，我们将4位KV缓存置于多轮智能体工作负载中，其中任务质量、缓存驻留和服务吞吐量必须联合衡量。其次，我们描述了使4位路径稳健所需的实际设计选择，包括非对称K/V处理、Walsh-Hadamard旋转、QJL移除和块尺度变体。第三，我们展示了在AMD GPU上的服务优化，包括优化的解码注意力核和UltraQuant——一种FP4近似路径，使用FP8查询、FP4 KV张量、UE8M0组尺度以及CDNA4上的原生scaled-MFMA支持。在长上下文、多轮智能体工作负载中，UltraQuant在缓存压力大的后期轮次中将P50首次令牌时间减少3.47倍（所有轮次平均2.3倍），并将输出吞吐量相对于FP8 KV基线提高1.63倍。

## Abstract
Context-heavy agents place unusual pressure on the key-value (KV) cache: long prefixes are reused across many short turns, while concurrency determines whether the serving system can keep GPUs utilized. We study 4-bit KV-cache compression for this setting, using TurboQuant-style rotation and codebook quantization as a quality anchor and vLLM FP8 KV caching as the deployment anchor. We report three contributions. First, we frame 4-bit KV caching around multi-round agent workloads where task quality, cache residency, and serving throughput must be measured jointly. Second, we describe the practical design choices needed to make the 4-bit path robust, including asymmetric K/V treatment, Walsh-Hadamard rotation, QJL removal, and block-scale variants. Third, we present serving optimizations on AMD GPUs, including optimized decode-attention kernels and UltraQuant, an FP4 approximation path that uses FP8 queries, FP4 KV tensors, UE8M0 group scales, and native scaled-MFMA support on CDNA4. On a long-context, multi-turn agentic workload, UltraQuant cuts P50 time-to-first-token by 3.47x in the cache-pressured late rounds (2.3x across all rounds) and raises output throughput by 1.63x over the FP8 KV baseline.

- **研究目标**：提出一种面向部署的 4-bit KV 缓存方案，在保持任务质量的同时，提升缓存驻留能力和服务吞吐，使端到端服务指标（TTFT、TPOT、输出吞吐量）超越 FP8 基线。

## 2. 方法论：核心思想、关键技术细节、算法流程

### 核心思想
- 采用 TurboQuant 风格的 Walsh-Hadamard 旋转将 KV 缓存分布近高斯化，然后使用 4-bit 量化。但用 **FP4 微张量近似**替代码本，使反量化由 GPU 矩阵核心（scaled-MFMA）原生完成，消除软件开销。
- 提出两个实现路径：**Ultra-TQ**（优化的 TurboQuant 代码本路径）和 **UltraQuant**（FP4 近似路径，最终部署选择）。

### 关键技术细节
1. **非对称 K/V 处理**：Keys 需旋转以降低异常值；Values 不旋转，直接量化。
2. **组级尺度（Group-32）**：将头维度按 32 通道分组，每组使用一个 UE8M0 指数尺度，组内值映射到 FP4 E2M1 网格。每组占用 16 字节 + 1 字节 = 17 字节（每元素 4.25 位）。
3. **全局常数优化（ConstOpt）**：通过单次前向传播在捕获的旋转 key 激活上，最小化重建误差 `E[(z - c/m * q(z/(c/m)))^2]`，得到全局常数 `c=0.156`。该常数模型无关，仅依赖头维度 `d`。这使得 FP4 网格更接近 Lloyd–Max 最优分布。
4. **去除每 token ℓ2 归一化**：因为组级 absmax 尺度已吸收幅度变化，无需额外存储 norm 值。

### 算法流程（文字说明）
**编码**：  
- 输入 BF16 K、V → 对 K 应用 Hadamard 旋转（V 不旋转）。  
- 将张量按 32 通道分组。  
- 对每组计算绝对最大值 `m`，再计算 `E = round(log2(c * m))` 得到 UE8M0 指数。  
- 每个元素舍入到最近的 FP4 水平，存储为每字节两码和每组的 UE8M0 字节。

**解码与注意力计算**：  
- 查询 Q 先经 Hadamard 旋转并转换为 FP8 E4M3。  
- 从缓存加载 FP4 码和 UE8M0 尺度；反量化公式：`x̂ = code * 2^(byte - 127)`。  
- QK 乘积通过 `v_mfma_scale_f32_16x16x128_f8f6f4` 指令一次性完成，无需显式反量化。  
- PV 路径类似，利用硬件低精度转换将 FP4 + UE8M0 转为 BF16，无需软件查找表。
