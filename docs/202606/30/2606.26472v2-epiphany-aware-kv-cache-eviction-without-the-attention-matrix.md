## 摘要
随着推理模型生成长达数万token的思维链，KV缓存日益成为部署瓶颈。现有的缓存淘汰方法通过注意力权重对token进行排序，但在长推理轨迹中这是一种噪声较大的重要性代理指标，并且强制模型实例化注意力矩阵，从而禁止在生产推理中使用融合内核。在本工作中，我们转而使用一种称为顿悟分数的指标对token进行评分：该分数直接从前向传播中读取模型内部表示的变化，无需注意力矩阵且额外状态可忽略。由此产生的缓存淘汰方法EpiKV无需训练、分类器或自定义内核，可直接在FlashAttention推理堆栈中保持不变地使用——其可行上下文长度可达基于注意力评分的16倍。我们发现，注意力权重在长推理轨迹中表现不佳，因为早期层随着token长度的增加而退化；我们量化了这种退化趋势（中间上层影响为负），并通过因果滚动z分数去除了位置趋势。在4096 token缓存下，EpiKV在MATH-500上达到72%，与最强的基于注意力的基线（ThinKV 71%，H2O 67%）持平；其延迟归一化KV变体在8192 token下，在AIME-2024上以最高2.8倍的速度达到37%，而最佳基线为33%。

## Abstract
As reasoning models emit chains of thought tens of thousands of tokens long, KV cache increasingly becomes a deployment bottleneck. Existing cache eviction methods rank tokens by attention weight, which is a noisy importance proxy in long reasoning traces, and prohibits the use of fused kernels in production inference by forcing the model to materialize the attention matrix. In this work, we instead score tokens with a metric we term the epiphany score: the change in the model's internal representation, read directly from the forward pass with no attention matrix and negligible extra state. Our resulting cache eviction method, EpiKV, requires no training, classifier, or custom kernel, and can be used directly in FlashAttention inference stacks unchanged -- scaling to a 16x longer feasible context than attention-based scoring. upper-mid layers negatively) and remove a positional trend with a causal rolling z-score. ## 1. 核心问题与整体含义（研究动机与背景）
- **问题**：推理模型（如DeepSeek-R1）生成长思维链（数万token），KV缓存线性增长成为部署瓶颈。现有KV缓存淘汰方法大多依赖注意力权重进行重要性评分，存在两大缺陷：
  - **重要性估计噪声大**：注意力汇点（sinks）吸收无关权重，填充token也吸引注意力，但实际并不重要。
  - **架构开销高**：读取注意力权重需物化 \(n \times n\) 注意力矩阵，迫使使用eager kernel，无法利用FlashAttention等融合内核。80GB A100在上下文超过8192 token时即显存溢出（OOM），而FlashAttention可扩展到65536 token。
- **核心目标**：设计一种**无需注意力矩阵**、**FlashAttention兼容**、**zero训练/分类器/自定义kernel**的缓存淘汰方法，在降低显存占用的同时匹配或超越基于注意力的淘汰精度。

## 2. 方法论：核心思想、关键技术细节
- **核心思想**：利用模型**前向传播过程中的内部表示变化**（hidden state L2 diff）作为token重要性的指标，称为“顿悟分数”（epiphany score）。关键发现是**不同层的表示变化与重要性呈相反的相关模式**：
  - **Band A（第7–13层）**：表示变化越大，token越重要（正相关）。这些层对应机制可解释性中事实检索与组合的位置。
  - **Band B（第18–25层）**：表示变化越大，token越不重要（负相关）。这些上层更多用于预测低信息量的流畅续写。
- **关键技术细节**：
  - **原始信号**：每层每个token的L2 diff \(g_l(t) = \|h_l(t) - h_l(t-1)\|_2\)。
  - **双带组合**：使用Band A和Band B的代表层（实验采用第10层和第21层）的滚动均值作差：\(s(t) = \bar{g}_{10}(t) - \bar{g}_{21}(t)\)。
  - **位置趋势校正**：原始信号在长序列中呈现单调变化（早期层下降、后期层上升），导致内容与位置混淆。采用**因果滚动z-score**去趋势：
    \[
    z_l(t) = \frac{g_l(t) - \mu_l(t)}{\sigma_l(t)+\varepsilon}
    \]
    其中窗口大小 \(w=64\)（仅用历史信息）。最终评分：\(z_{10}(t) - z_{21}(t)\)。
  - **计算方法**：评分仅需读取forward pass中`output_hidden_states`接口暴露的hidden states和KV缓存本身，无需`output_attentions`，因此完全兼容FlashAttention，无额外显存开销。
- **额外变体**：还有直接利用KV向量统计（key variance、value variance、Lag-KV（借鉴LagKV的延迟归一化但在线版本））的注意力无关方法。
