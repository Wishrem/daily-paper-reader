## 摘要
多模态大语言模型依赖上下文学习进行快速任务适应，但其可扩展性受到有限上下文窗口和长多模态序列中键值缓存成本不断增长的严重限制。现有的内存压缩方法通常依赖于刚性令牌移除或样本相关的重要性估计，这引入了偏差，破坏了语义结构（特别是对于视觉表示），并产生无法适应新查询的静态内存。我们提出TASM（任务感知结构化内存），一种无需训练的框架，通过任务感知、结构保持和动态可访问的内存构建来解决这些限制。TASM采用任务向量引导压缩，用捕获演示中共享相关性的任务级方向替换样本特定信号。为了保持底层流形，它通过二分图匹配实现语义感知的令牌合并，聚合令牌而不进行破坏性剪枝。最后，TASM将内存结构化为包括紧凑核心内存和潜在库的层次结构，促进查询自适应的动态检索。评估确认TASM在重度压缩下保持高性能，有效平衡效率与适应性。

## Abstract
Multi-modal large language models (MLLMs) depend on in-context learning (ICL) for rapid task adaptation, but their scalability is severely limited by finite context windows and the growing cost of key-value (KV) caches in long multi-modal sequences. Existing memory compression approaches typically rely on rigid token removal or sample-dependent importance estimation, which introduces bias, disrupts semantic structure, particularly for visual representations, and yields static memories that cannot adapt to new queries. We introduce TASM (Task-Aware Structured Memory), a training-free framework that addresses these limitations through task-aware, structure-preserving, and dynamically accessible memory construction. TASM employs task-vector guided compression to replace sample-specific signals with a task-level direction that captures shared relevance across demonstrations. To preserve the underlying manifold, it applies semantics-aware token merging via bipartite graph matching, aggregating tokens without destructive pruning. Finally, TASM structures memory into a hierarchy comprising a compact Core Memory and a Latent Bank, facilitating query-adaptive dynamic retrieval. Evaluations confirm TASM maintains high performance under heavy compression, effectively balancing efficiency with adaptability.

#### 1. 核心问题与整体含义（研究动机和背景）
- **核心问题**：多模态大语言模型（MLLMs）依赖上下文学习（ICL）进行快速任务适应，但在长多模态序列（如图像+文本）中，LLM 的有限上下文窗口和不断增长的键值（KV）缓存导致显存爆炸和计算开销，严重限制了可扩展性。
- **现有局限**：现有内存压缩方法（如硬剪枝、基于注意力分数的样本级重要性估计）存在三个结构性问题：
  - **样本特异性偏差**：依赖特定演示的注意力分数，可能移除对测试查询关键的信息。
  - **拓扑破坏**：硬剪枝丢弃视觉令牌，破坏空间和语义结构，损害空间定位任务。
  - **静态记忆僵化**：压缩后的缓存静态不变，无法适应复杂查询所需的精细细节。
- **本文目标**：提出一种无需训练、任务感知、结构保持且动态可访问的内存压缩框架（TASM），以平衡效率与适应性，使多模态ICL能在消费级硬件上高效运行。

#### 2. 方法论：核心思想、关键技术细节
- **核心思想**：将KV缓存重构为表示任务的层次化结构化记忆，而非静态缓冲区。通过任务向量引导压缩、语义感知令牌合并和查询自适应动态检索实现。
- **关键技术细节**：
  1. **任务向量引导的重要性估计**：
     - 从少量示例中提取任务向量 τ，表示从问题到答案的变换方向（通过计算问答对表示的质心差并归一化）。
     - 将每个令牌的键表示投影到 τ 上，结合值范数，得到任务感知重要性分数 `s_task`。
     - 引入层自适应门控 λ(l)：浅层侧重局部注意力（保留语法/边缘），深层侧重任务向量（全局语义），通过移位sigmoid平滑过渡。
  2. **语义感知令牌合并**：
     - 将令牌分为“汇点”（高分保留）和“源点”（低分待合并），构建二分图。
     - 定义加权兼容性度量，对视觉令牌添加空间约束（只能合并邻域窗口内的令牌），强制保持2D流形。
     - 通过加权质心更新汇点表示，实现信息聚合而非丢弃。
  3. **信息几何动态检索**：
     - 层次化内存：核心记忆（Core Memory，GPU常驻压缩缓存）和潜在库（Latent Bank，CPU卸载的高分辨率令牌）。
     - 基于 Jensen-Shannon 散度监测注意力分布变化，当检测到分布漂移（超过阈值 δ）时，从潜在库检索 top-k 相关令牌，动态扩展当前缓存。
- **算法流程**（见论文伪代码）：初始化任务向量 → 对当前 token 计算注意力分布并判断 JS 散度 → 若触发则检索潜在库 → 重要性评分 → 语义合并 → 更新核心记忆和潜在库。

#### 6. 主要结论与发现
- TASM 在所有基准上**显著优于**现有压缩方法（EMLoC等），且在全上下文MLoC性能基础上进一步**提升**（如YouCook2达到109.5 vs MLoC 108.8），同时内存减少78.6%（ImageNet-100从16264 tokens 降至3485）。

- **任务向量具有零样本迁移能力**：在未见过的类别上仅下降0.8%（64.2% vs Oracle 65.0%）。
- **跨架构泛化强**：在LLaVA-NeXT和InternVL3上性能损失极小（-0.4% vs 全上下文），而SnapKV和EMLoC损失更大（尤其InternVL3的-8.3%）。

#### 7. 优点
- **无需训练**：直接作用于预训练MLLM，无参数更新，计算成本低。
- **结构保持**：软合并替代硬剪枝，保留2D空间拓扑，适合多模态视觉任务。
- **动态适应性**：根据查询复杂度自动检索历史信息，避免静态内存僵化。
- **任务导向**：利用任务向量从全局角度评估令牌重要性，减少样本偏差和注意力下沉影响。
- **高效可扩展**：内存与计算复杂度接近线性（O(T·(N_core+N_ret))），支持极长上下文（32k序列仅11.5GB显存）。
- **广泛适用**：已验证多种基础模型（Qwen2/2.5-VL、LLaVA、InternVL）和多种任务类型。

#### 8. 不足与局限
- **依赖少量示例提取任务向量**：虽然样本效率高（4-shot即可），但需要存在合适的少样本示例，且任务向量可能仅在任务内泛化（跨任务适用性未验证）。
- **超参数较多**：需要调优层门控参数（α,β,κ）、JS阈值δ、空间窗口大小等，虽然论文给出默认值并分析了灵敏度，但在新场景下仍需尝试。
- **二分图匹配增加预填充开销**：在长序列上匹配耗时可能成为瓶颈（论文报告预填充时间比EMLoC高约35ms@32k），虽然可接受。
- **动态检索依赖GPU-CPU通信**：潜在库卸载至CPU，检索时需数据传输，可能引入额外延迟（论文未详细分析通信带宽影响）。
- **未讨论极大上下文（>32k）**：实验最大32k，更极端的场景（100k tokens）未测试。
- **公平性备注**：对比方法中EMLoC等可能未达到最优超参数设置，但论文已尽量匹配压缩率（20%预算）。消融实验中“硬剪枝基线”未明确是哪种方法（可能是Top-K注意力剪枝），对比标准稍有模糊。
