## 摘要
稀疏专家混合（MoE）架构通过仅基于 top-$k$ 路由激活一小部分专家，在固定推理预算下实现大语言模型参数扩展。虽然这保持了因果性并适用于自回归语言模型，但离散的 top-$k$ 算子不可微分，迫使每个输入具有固定数量的活跃专家，导致计算利用效率低下。我们提出 SoftMoE，它用截断的软 top-$k$ LapSum 松弛替代离散路由，从而实现对专家路由的梯度优化。我们进一步参数化每层活跃专家的平均数量并施加全局预算约束，使模型能够学习如何跨层分配专家容量。SoftMoE 完全兼容自回归建模，在语言建模和下游任务上达到与稀疏 MoE 相当或更优的性能，同时激活显著更少的专家。值得注意的是，学习到的分配高度不均匀，较晚的层激活更多专家。源代码已公开提供$^\dagger$。

## Abstract
Sparse Mixture-of-Experts (MoE) architectures enable scaling LLM parameters under a fixed inference budget by activating only a small subset of experts via top-$k$ routing. While this preserves causality and suits autoregressive language models, the discrete top-$k$ operator is not differentiable, forcing a fixed number of active experts per input and resulting in inefficient use of computation. We propose SoftMoE, which replaces discrete routing with a truncated soft top-$k$ LapSum relaxation, allowing gradient-based optimization of expert routing. We further parameterize the mean number of active experts per layer and impose a global budget constraint, enabling the model to learn how to allocate expert capacity across layers. SoftMoE remains fully compatible with autoregressive modeling and achieves performance comparable to or better than sparse MoE on language modeling and downstream tasks, while activating significantly fewer experts. Notably, the learned allocation is highly non-uniform, with later layers activating more experts. The source code is publicly available$^\dagger$.

## 1. 核心问题与整体含义

大型语言模型的扩展受限于推理成本，稀疏专家混合（Mixture-of-Experts, MoE）架构通过只激活每个输入的一小部分专家（即 top-\(k\) 路由）来缓解这一问题。然而，标准稀疏 MoE 使用的离散 top-\(k\) 算子不可微，导致训练时梯度无法通过路由选择过程传播，必须预先固定每层激活的专家数量 \(k\)，无法根据输入或层自适应调整。这种固定的计算分配造成了计算资源的低效使用，模型不能在低复杂度层节省计算量，也无法在高复杂度层增加容量。

本文提出 **SoftMoE**，用基于 LapSum 松弛的可微 soft top-\(k\) 算子替代离散路由，使得路由选择本身可以端到端梯度优化；同时通过全局预算约束和参数化每层期望活跃专家数，让模型自动学习跨层专家容量分配。SoftMoE 在保持自回归因果性和计算效率的前提下，实现了与稀疏 MoE 相当或更优的性能，同时激活显著更少的专家，为大规模语言模型的高效扩展提供了新范式。

## 2. 方法论

### 核心思想
- **可微 soft top-\(k\) 路由**：利用 LapSum 函数（Laplace 累积分布函数之和）构造一个平滑的 top-\(k\) 近似。给定路由分数 \(\mathbf{r} \in \mathbb{R}^n\)，求解方程 \(\sum_{i=1}^n F_{\text{Lap}}(r_i - x) = k\) 得到连续阈值 \(\tilde{x}\)，则每个专家的软权重为 \( \tilde{p}_i = F_{\text{Lap}}(r_i - \tilde{x}) \in [0,1] \)，且满足 \(\sum_i \tilde{p}_i = k\)。该操作可微，且计算复杂度 \(O(n)\)。
- **截断操作**：为了保持计算稀疏性，对软权重应用截断 \(\mathcal{T}(z) = z \cdot \mathbb{I}_{z > \tau}\)，移除低于阈值 \(\tau\) 的贡献。梯度在激活专家处通过，未激活专家梯度为零。这使得每 token 活跃专家数变为输入依赖的，而非固定。
- **可学习的层间专家分配**：对每层引入可学习参数 \(k_l\)（该层期望活跃专家数），通过 softmax 重参数化满足约束 \(k_l \ge 1\) 且 \(\sum_{l=1}^L k_l = K\)（全局预算 \(K\)）。参数 \(\eta_l\) 无约束，经 \(\pi = \text{softmax}(\eta)\) 和 \(k_l = \pi_l \cdot (K - L) + 1\) 映射到可行域。这使得模型能用梯度下降联合优化专家分配。

### 关键技术细节
- 路由网络：\(G(x) = \mathcal{T}(\text{SoftTopK}(x W_g, k_l))\)。
- 单层输出：\(y = \sum_{i=1}^n G(x)_i \cdot E_i(x)\)，其中 \(E_i\) 为第 \(i\) 个专家网络。
- 全局预算超参数 \(K\)：例如 \(K = 2L\) 对应平均每层 2 个专家，其计算量与 standard sparse MoE-k=2 相当。
- 辅助损失：使用负载均衡损失（Load Balancing Loss）防止专家极端不平衡。

### 算法流程（文字描述）
1. 给定输入 token 表示 \(x\)，通过可学习门控矩阵 \(W_g\) 计算路由分数 \(r = xW_g\)。
2. 对分数应用 SoftTopK（LapSum 松弛），得到连续权重 \(\tilde{p}\)，其和等于当前层期望数 \(k_l\)。
3. 对 \(\tilde{p}\) 进行阈值截断，保留大于 \(\tau\) 的部分，得到稀疏掩码 \(m\)。
4. 每个专家 \(E_i\) 对 \(x\) 计算输出，然后按 \(m_i\) 加权求和得到层输出。
5. 训练时，通过反向传播更新门控矩阵 \(W_g\)、专家参数以及分配参数 \(\eta_l\) 。
6. 分配参数 \(\eta_l\) 经过 softmax + 仿射变换保证全局预算约束和每层不少于 1 个专家。

## 3. 实验设计

### 数据集与场景
- **预训练语料**：
  - Open Web Text (OWT)：约 9B tokens，BPE tokenizer，词汇表 32k。
  - Common Crawl (C4)：约 206B tokens，同样 tokenizer。
  各用 98% 训练，2% 验证。
- **下游评估基准**：
  - PIQA（物理常识推理）
  - HellaSwag（句子补全）
  - ARC-E（基础科学问答）
  均为零样本评估。

### 对比方法
- **Sparse MoE**（Switch Transformer 架构）：\(k = 1, 2, 3, 4\)，无 jitter（因为 jitter 会降低性能，见消融实验）。
- **SoftMoE\***：使用 soft top-\(k\) 路由但固定每层 \(k\)，不学习层间分配。
- **SoftMoE**：完整可学习层间分配，设定不同起始 \(k\) 和截断扩展因子 \(\alpha\)。

### 模型配置
- 10 层 decoder-only GPT-2 架构，32 专家/层，每专家 5M 参数，总参数量 1.63B。
- 训练步数：OWT 18k 步，C4 164k 步。
- 批次大小：OWT 240，C4 600；序列长度 2048。
- 优化器：Adam (β1=0.9, β2=0.95)；学习率 cosine decay 从 1e-3 到 1e-5；预热 1% 步数。
- 使用辅助负载均衡损失，温度从 5 退火到 1。

## 6. 主要结论与发现

1. **SoftMoE 效率更高**：在相同或更低预算下，SoftMoE 的验证损失低于或等于 Sparse MoE。例如表 1 中，OWT 上 SoftMoE\*-1.5 损失 2.78（vs Sparse MoE-2 的 2.79），训练活跃专家平均仅 1.53（vs 2.0）。
2. **学习到的层间分配高度不均匀**：后期层（最后 3 层）获得约 50% 的预算，前期层分配更少（图 1、6）。这揭示了深度语言模型在后期需要更多专家容量（可能对应语义级别处理）。
3. **下游任务性能优异**：在 HellaSwag 和 PIQA 上，SoftMoE 普遍优于或持平 Sparse MoE，只在 ARC-E 上略有波动。
4. **可微路由不引起训练不稳定**：训练曲线平滑，无显著波动（图 4）。
5. **负载均衡维持良好**：专家负载方差与 Sparse MoE 相当（表 5）。

## 7. 优点

- **新颖性**：首次将可微 top-\(k\) 松弛（LapSum）成功应用于大规模 MoE 语言模型，并实现层间计算预算的学习。
- **兼容性**：保持 token 独立处理和自回归因果性，避免 token 混合等破坏因果的做法，可直接替代现有 MoE 层的路由。
- **低开销**：LapSum 计算 \(O(n)\)，且截断后实际激活专家数少；整体增加的计算量远小于专家前向的节省。
- **自适应能力**：模型自动为不同层分配不同容量，从而在固定总计算下获得更好的性能。
- **简洁的数学形式**：通过 softmax + 仿射变换优雅地实现预算约束的可微参数化。

## 8. 不足与局限

- **规模验证不足**：仅在 1.63B 参数模型上实验，未在更大规模（如 100B+）验证，结论推广需谨慎。作者在 Limitations 中也承认了这一点。
- **语言覆盖有限**：仅使用英文语料（OWT 和 C4），未评估多语言或多模态情况，可能导致结论偏倚。
- **下游任务较少**：仅 3 个零样本 benchmark，缺少对话、代码、长文本等更全面的评估。
- **截断阈值 \(\tau\) 需要统计初始化**：需要从初始批次收集统计数据调整阈值，增加了调参维度（表 6 显示不同 \(\tau\) 影响性能）。
- **计算资源报告不够详细**：虽然提供了 GPU 时数，但未比较实际 FLOPs（稀疏 MoE 在恒定激活数下可更高效利用硬件），也未讨论内存开销。
- **是否真的能在实际部署中加速**：由于不规则的激活专家数可能导致系统负载不均衡或 batch 调度难度，论文未讨论分布式推理场景下的实际加速效果。
