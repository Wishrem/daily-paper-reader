## 摘要
为AI加速器编写高性能内核需要深入的专业知识，涉及分块、指令选择、数据布局和算子融合，这给程序员带来了沉重负担。在本文中，我们专注于基于分块的AI加速器程序，并提出了Axon，一个面向张量程序的综合超优化器：它利用程序综合技术从语义规范自动生成目标指令，并探索语义等价的程序变体，通过经验选择性能最佳的内核。Axon通过计算图传播算子来发现代数变换，并使用无界张量上的SMT求解器来保证所有变换保持语义，无需手工编写重写规则。然后它将张量操作降级为目标ISA指令，探索由硬件描述约束的分块配置，并融合算子和指令以最小化内存流量。

## Abstract
Writing high performance kernels for AI accelerators requires deep expertise in tiling, instruction selection, data layout, and operator fusion placing a significant burden on programmers. In this paper, we focus on tile based AI accelerator programs and present Axon, a synthesizing superoptimizer for tensor programs: it uses program synthesis to automatically generate target instructions from semantics specifications, and explores semantically equivalent program variants to select the best performing kernel empirically. Axon discovers algebraic transformations by propagating operators through computation graphs and uses SMT over unbounded tensors to guarantee that all transformations preserve semantics without requiring hand crafted rewrite rules. It then lowers tensor operations to target ISA instructions, explores tiling configurations constrained by hardware descriptions, and fuses operators and instructions to minimize memory traffic.

## 2. 论文提出的方法论：核心思想、关键技术细节

### 整体流程
Axon将输入张量程序逐步优化为最终内核，所有变体在统一数据结构 **ν-Graph（nu-graph）** 中同时维护，最后通过实验选择最佳方案。

主要阶段：
1. **代数变换**：通过**算子传播算法**（Algorithm 1）尝试将算子向下移动（reorder），使用**SMT求解器**在无界张量上证明交换后语义等价，若不成立则保留下方算子。该算法自动发现等价图变体（如RMSNorm中的`multiply`和`matmul`交换），无需手工规则。
2. **符号分块（Tiling）**：引入三个层次的符号参数（strip、block、tile维度），受硬件约束（如每个引擎的最大维度）。各分支可独立分块，最大化数据复用。
3. **ISA合成**：通过**sketch-driven程序合成**（Algorithm 2）将张量操作分解为目标ISA指令。首先筛选共享子操作的指令池，然后通过填充洞（holes）生成指令序列，并用SMT验证等价性。例如`matmul(x,y)`可合成`nc_matmul(nc_transpose(x), y)`。
4. **融合**：在ISA级别再次进行代数变换（发现新的交换机会），然后进行两种融合：
   - **指令融合**：将连续指令替换为单一等价指令（如`dma_copy`+`nc_transpose`→`dma_transpose`）。
   - **算子/循环融合**：将相同条带维度的相邻节点合并，减少HBM流量。
5. **代码生成与选择**：实例化符号参数，发出具体NKI程序；对可编译候选执行并测量延迟，选择性能最佳内核，同时用随机输入验证数值正确性（rtol=1e-4, atol=1e-4）。

### 关键技术细节
- **ν-Graph**：一组语义等价的完整数据流图变体，每个变体携带元数据（分块参数、ISA指令、引擎约束等），通过各阶段不断扩展。
- **等价性检查**：将张量操作用SMT编码为形状条件和计算条件（基于无界张量上的算术公理），对非线性函数作为未解释函数保守拒绝。
- **硬件约束**：指令语义包含有效分块维度范围（如`nc_matmul`限制K≤128, M≤128, N≤512），用于约束分块参数。

## 6. 论文的主要结论与发现

- Axon在独立算子上相比Neuron编译器**最高加速3.7倍**（SiLU），几何平均加速1.23x–1.90x。
- 在多算子内核上**最高加速19倍**（Transpose+MatMul），几何平均1.08x–1.32x。
- 相比手动优化NKI内核，Axon**最高提升1.35倍**（Cumsum几何平均1.07x, RoPE 1.06x）。
- 相比Mirage，在支持的基准上**几何平均加速2–10%**，且能处理Mirage无法处理的16个额外基准。
- 消融实验显示：**算子融合贡献最大**（禁用后几何平均慢35–50%），**代数变换贡献15–35%**，两者互补。
- 编译开销合理（最复杂内核约1.7小时），是一次性成本。

## 7. 优点

1. **高度自动化**：从高级NumPy-like输入到优化内核，无需手工规则或模板。
2. **正确性保证**：使用SMT在无界张量上证明变换等价（强于随机测试），非线性函数保守拒绝，避免错误。
3. **统一表示**：ν-Graph同时维护所有变体，避免过早决策，便于探索组合优化。
4. **硬件感知**：指令合成和分块直接利用硬件约束（引擎限制、缓冲容量）。
5. **超越现有方法**：在多种基准上超越生产级编译器（Neuron）和专有搜索编译器（Mirage），且能达到甚至超过手工代码。

## 8. 不足与局限

1. **架构特定性**：当前仅针对Trainium（多引擎、DMA驱动、tile-based），移植到其他加速器（如TPU、GPU、MTIA）需要新ISA语义和代码发射器，核心算法可复用但需调整。
2. **浮点精度**：等价性检查使用实数理论忽略浮点误差；后续通过随机测试验证，但无正式保证。
3. **非线性函数受限**：如sigmoid、exp被当作未解释函数，可能错过一些合法交换机会（但保证了安全性）。
4. **搜索开销**：最复杂内核编译时间约1.7小时，虽为一次性成本，但对快速迭代仍有影响。
5. **评估局限**：
   - 仅单核，未评估多核或端到端推理。
   - 未覆盖卷积、稀疏算子、训练反向传播等常见模式。
   - 仅与Mirage三个基准可比，无法完全证明普遍优势。
6. **不可编译候选**：约24–40%的候选因缓冲容量不足被拒绝，Axon无直接控制，依赖后端。
