## 摘要
最近的手机级移动SoC为设备端视觉-语言模型（VLM）推理提供了实用的NPU执行路径，但开发者仍然缺乏用于在异构后端上映射VLM流水线的阶段级指导。我们对高通SM8750（骁龙8 Elite）上的VLM推理进行了硬件在环表征，涵盖了阶段吞吐量、缓存状态效应、100次运行的热稳定性、能耗、异构CPU/NPU流水线配置以及视觉令牌预算敏感性。通过使用FastVLM-0.5B作为端到端案例研究，并结合四个架构家族的编码器专用测量，我们展示了阶段的重要性：NPU执行高度依赖于阶段，预填充阶段提供了1.64倍的加速，但解码阶段仅为1.18倍，而视觉编码器相对于CPU实现了20-45倍的加速。这些优势转化为稳态温度降低10.47°C和能耗降低2.52倍，从而在始终开启的设置中避免了热节流。最后，我们展示了四步图重写使先前不支持的编码器（如Phi-3.5-V）能够达到QNN路径，加速高达22倍，为移动VLM部署提供了实用的移植方案。

## Abstract
Recent phone-class mobile SoCs expose practical NPU execution paths for on-device vision-language model (VLM) inference, but developers still lack phase-level guidance for mapping VLM pipelines across heterogeneous backends. We present a hardware-in-the-loop characterization of VLM inference on the Qualcomm SM8750 (Snapdragon 8 Elite), covering phase throughput, cache-state effects, 100-run thermal stability, energy, heterogeneous CPU/NPU pipeline configurations, and visual-token-budget sensitivity. Using FastVLM-0.5B as an end-to-end case study, together with encoder-only measurements across four architecture families, we show that phase matters: NPU execution is highly phase-dependent, delivering 1.64x speedup for prefill but only 1.18x for decode, while vision encoders achieve 20-45x speedups over CPU. These gains translate into 10.47 degrees C lower steady-state temperature and 2.52x lower energy, avoiding thermal throttling in always-on settings. Finally, we show that a four-step graph rewrite enables previously unsupported encoders, such as Phi-3.5-V, to reach the QNN path with up to 22x speedup, providing a practical porting recipe for mobile VLM deployment.

- **研究动机**：尽管现代手机级移动SoC（如高通SM8750 Snapdragon 8 Elite）已为设备端视觉语言模型（VLM）推理提供实用的NPU执行路径，但开发者缺乏**阶段级（phase-level）** 指导来将VLM流水线（编码、预填充、解码）映射到异构后端（CPU/NPU/GPU）。现有工作多针对纯文本LLM，缺少对视觉编码阶段、热行为以及编码器移植路径的系统研究。
- **整体含义**：VLM推理的三个阶段具有根本不同的硬件亲和性：预填充（compute-bound，NPU优势大）、解码（bandwidth-bound，CPU更节能）、视觉编码（static-shape，NPU效率最高）。统一对待这些阶段会导致延迟、能量和热预算的浪费。论文旨在通过硬件在环表征，提供阶段感知的部署指南和实用的QNN移植工作流。

## 2. 论文提出的方法论：核心思想、关键技术细节

- **核心思想**：对VLM推理进行**阶段分解**，分别测量每个阶段的吞吐量、缓存状态效应、热稳定性、能耗，以及异构流水线配置和视觉token预算敏感性。提出了一种**四步图重写方法**，使原本不支持的编码器（如Phi-3.5-V）能够运行在QNN NPU路径上。
- **关键技术细节**：
  - **阶段分解**：根据Roofline分析，将VLM推理分为视觉编码（静态-shape、计算密集）、预填充（高运算强度，大batch GEMM）、解码（逐token，KV缓存带宽受限）。
  - **缓存状态分类**：定义了三种状态——S1（冷启动：进程杀死+缓存清除+QNN上下文缓存删除）、S2（cold-cache：进程重启但模型热缓存，QNN上下文缓存完好）、S3（warm：进程存活，QNN上下文在VTCM中，仅分派）。
  - **硬件平台**：高通SM8750，3nm SoC，Hexagon NPU（标量/向量/张量引擎+VTCM），Adreno GPU，Oryon CPU 4.32GHz，LPDDR5x 5300MHz。
  - **软件栈**：CPU路径使用LiteRT + XNNPACK FP16；NPU路径使用LiteRT + Qualcomm AI Engine delegate（QNN）；W4A8量化路径使用QAIRT（INT4权重、INT8激活）。
  - **四步图重写方法**（针对Phi-3.5-V等不支持编码器）：
    1. 静态形状导出（将动态符号维度替换为固定分辨率常数）；
    2. 注意力分解（将Flash Attention/SDPA重写为显式Q/K/V矩阵乘法）；
    3. 显式Cast插入（在精度边界插入Cast节点，如FP32投影头与INT8编码体之间）；
    4. CPU回退分区（使用QNN分区模型API，将少量不支持算子（如自定义旋转位置嵌入、GatherElements）留在CPU上，保持96.8% FLOPs在Hexagon上）。

## 6. 论文的主要结论与发现

1. **NPU加速高度依赖阶段**：预填充1.64×，解码仅1.18×，视觉编码器20-45×（如ViT-B/16为44.5×），GPU为NPU的1.7-6.9倍（GPU不是NPU的竞争替代）。
2. **NPU热优势显著**：100次连续推理，NPU路径稳态温度40.78°C（峰值44.20°C），CPU路径51.25°C（峰值55.30°C），低10.47°C，方差低5倍；CPU在15分钟持续负载下达到82°C触发节流，NPU仅66°C且无节流。
3. **能源效率**：NPU每次请求能耗低2.52倍。
4. **冷启动开销**：NPU执行初始化（S2状态）需要536ms（CPU仅84ms），高达6.41倍开销。但热缓存下（S3）NPU仍保持1.4-1.6×优势，因此需持久化工人进程。
5. **异构流水线推荐**：Hybrid-PD（编码器+解码器CPU，预填充NPU）在冷启动下表现最佳（6,240ms冷、2,640ms热），兼顾冷/热平衡；NPU-only冷启动极差（26,820ms）。
6. **视觉token预算作为无重编译延迟旋钮**：从24到224 tokens，TTFS从0.361s线性增长到0.850s，解码吞吐稳定在95-100 tok/s，W4A8混合配置在64 tokens时TTFS 0.497s、97.5 tok/s。
7. **编码器架构选择比后端选择更重要**：NPU上最慢编码器（Phi-3.5-V 103.5ms）是 最快编码器（NanoVLM 14.3ms）的7倍，大于任何后端切换收益。
8. **四步图重写成功移植Phi-3.5-V**：从0% NPU FLOPs到96.8%，延迟从2286ms降至103.5ms（22.1×加速）。
9. **输出语义等价**：NPU INT8与CPU FP16在CIDEr和VQAv2上差异极小（0.4点和0.3%），部署级可接受。
