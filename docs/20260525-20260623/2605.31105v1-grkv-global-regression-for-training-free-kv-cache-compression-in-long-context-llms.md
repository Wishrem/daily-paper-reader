## 摘要
具有扩展上下文长度的大语言模型依赖键值（KV）缓存来支持对先前token的注意力计算。然而，维护KV缓存会带来显著的内存开销，这促使了通过驱逐和合并来实施固定预算的KV缓存压缩方法。现代驱逐方法越来越多地采用基于跨度的保留，因为保留连续跨度在经验上有效且更好地保持语义连贯性。然而，当与驱逐后合并相结合时，基于跨度的保留会将合并集中到一小部分跨度边界载体token上，产生高度不平衡的合并模式，加剧过度合并并增加信息损失。为解决这种不平衡，我们提出GRKV（KV缓存的全局回归），一种免训练的KV缓存合并方法，直接最小化压缩缓存与完整缓存注意力输出之间的差异。GRKV使用基于岭回归的合并步骤将驱逐token的信息分布到保留token上，同时正则化更新以防止过度平滑。在LongBench和RULER长上下文基准测试中，GRKV是唯一能以最小开销提升整体性能的合并方法。

## Abstract
Large language models (LLMs) with extended context lengths rely on the key-value (KV) cache to support attention over prior tokens. However, maintaining the KV cache incurs substantial memory overhead, motivating KV-cache compression methods that enforce a fixed budget through eviction and merging. Modern eviction methods increasingly adopt span-based retention because preserving contiguous spans is empirically effective and better preserves semantic coherence. Yet, when combined with post-eviction merging, span-based retention concentrates merges onto a small set of span-boundary carrier tokens, producing a highly imbalanced merge pattern that exacerbates over-merging and increases information loss. To address this imbalance, we propose GRKV (Global Regression for KV Cache), a training-free KV-cache merging method that directly minimizes the discrepancy between compressed-cache and full-cache attention outputs. GRKV uses ridge-regression-based merge steps to distribute information from evicted tokens across retained tokens, while regularizing the updates to prevent over-smoothing. Across the LongBench and RULER long-context benchmarks, GRKV is the only merging method that improves overall performance with minimal overhead.

## 1. 核心问题与整体含义（研究动机和背景）

- **问题**：大语言模型（LLMs）在处理长上下文时依赖 KV 缓存来存储先前 token 的键值对，但 KV 缓存占用大量显存。为降低内存开销，研究者提出通过“驱逐 + 合并”的方式对 KV 缓存进行压缩，即在固定预算下保留部分 token（驱逐），并将被驱逐 token 的信息合并到保留 token 中。
- **现有方法的局限性**：现代驱逐方法（如 SnapKV）逐渐采用“基于连续跨度”的保留策略（span-based retention），这种方式能更好地保持语义完整性。然而，当后续合并方法（如 CaM、D2O、AsymKV）仍采用**局部启发式匹配**（基于邻接或键相似性）时，被驱逐的 token 会大量集中在跨度边界上的一小部分“载体 token”（carrier tokens）上，导致：
  - 合并模式极度不平衡（imbalanced merge pattern），
  - 边界载体 token 被过度合并（over-merging），使其原始语义模糊甚至消失，
  - 整体信息丢失加剧，性能下降。
- **本文目标**：提出一种**免训练的全局合并方法** GRKV，通过直接最小化压缩缓存与完整缓存注意力输出之间的差异，将被驱逐 token 的信息分布到所有保留 token 上，从而缓解过合并问题，并保持生成质量。

## 2. 方法论：核心思想、关键技术细节

### 2.1 核心思想
- GRKV 将 KV 缓存合并视为一个**全局回归问题**：给定一个查询窗口（surrogate query window），优化保留的 K/V 矩阵，使得压缩缓存产生的注意力输出尽可能接近完整缓存的输出。
- 使用**岭回归（ridge regression）** 对保留的 K/V 进行更新，同时通过正则化项防止更新过于剧烈（over-smoothing）。
- 将被驱逐 token 的信息**均匀分布到所有保留 token**（而非仅边界 token），从而扩大载体容量，减轻过合并。

## 6. 主要结论与发现

- **GRKV 是唯一在所有设置下提升整体性能的合并方法**：在 LongBench 和 RULER 上，GRKV 始终改善基座驱逐方法的平均分，且优于基座的任务数最多（Llama 上 14/16，Mistral 上 12–14/16，RULER 上 8–10/13）。

## 7. 优点

1. **全局优化视角**：首次将合并问题建模为全局回归，直接对齐注意力输出，而非依赖局部启发式。
2. **有效缓解过合并**：通过将信息分布到所有保留 token，并用正则化限制更新幅度，避免了边界 token 被过度负载。
3. **免训练且即插即用**：不依赖任何微调或训练，可作为插件与现有任何驱逐方法结合。
4. **数学优雅且高效**：键/值步骤均推导出对偶形式，将大矩阵求逆转化为小规模（m×m 或 md×md）问题，实际运行效率高。
5. **广泛兼容**：在 Span-based（SnapKV、CriticalKV、PyramidKV、Ada-KV）和 Token-based（H2O）驱逐方法上均有效，在 7B 到 14B 模型上验证。
6. **消融充分**：对各个超参数进行了系统分析，结论可靠。

## 8. 不足与局限

1. **实验模型范围有限**：仅测试了 7B–14B 的开源模型，未验证更大模型（如 70B、70B+）或闭源模型（如 GPT-4、Claude）。更大模型的行为可能不同，正则化参数可能需要调整。
