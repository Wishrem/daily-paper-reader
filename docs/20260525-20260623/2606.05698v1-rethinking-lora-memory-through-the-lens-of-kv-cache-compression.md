## 摘要
参数化检索增强将文档信息编码到轻量级、文档特定的模块（如LoRA适配器）中，减少了将所有证据作为输入上下文的需求。然而，这种参数侧记忆如何与存储在KV缓存中的上下文侧记忆相互作用仍不清楚。我们通过在文档级问答中逐步驱逐文档键值状态并测量文档LoRA何时在保留上下文之外贡献，来研究这种相互作用。我们发现，当KV缓存大致完整时，文档LoRA贡献很小，但在激进压缩下变得越来越有用，当没有文档上下文保留时，恢复了13-21个ROUGE-L点。当基础模型编码文档，且适配器仅在答案生成期间应用时，增益最大，这表明文档LoRA更适合被理解为解码时的参数记忆，而非文档编码器。最后，问答式监督产生的适配器比原始上下文的下一个词预测要强得多。这些结果将文档LoRA定位为一种互补的记忆通道，其价值恰好在上下文侧证据稀缺时显现。

## Abstract
Parametric retrieval augmentation encodes document information into lightweight, document-specific modules such as LoRA adapters, reducing the need to include all evidence as input context. However, it remains unclear how this parameter-side memory interacts with context-side memory stored in the KV cache. We study this interaction in document-level question answering by progressively evicting document key-value states and measuring when a document LoRA contributes beyond the retained context. We find that document LoRA adds little when the KV cache is largely intact, but becomes increasingly useful under aggressive compression, recovering 13-21 ROUGE-L points when no document context remains. The gain is largest when the base model encodes the document, and the adapter is applied only during answer generation, suggesting that document LoRA is better understood as decoding-time parametric memory than as a document encoder. Finally, QA-style supervision produces substantially stronger adapters than raw-context next-token-prediction. These results position document LoRA as a complementary memory channel whose value emerges precisely when context-side evidence is scarce.

- **研究动机**：检索增强生成（RAG）通常将检索到的文档附加到输入上下文中，但长文档导致KV缓存占用大量内存。已有两类工作试图减少对完整文档KV缓存的依赖：一是KV缓存压缩（例如驱逐不重要的键值对）；二是参数化检索增强，将文档信息编码为轻量级、文档特定的LoRA适配器。然而，这两类方法通常被孤立研究，它们之间的相互作用（参数侧记忆与上下文侧记忆如何互补或竞争）尚未被充分探索。
- **核心问题**：在不同KV缓存压缩程度下，文档LoRA（参数记忆）何时以及如何提供额外价值？LoRA最佳的应用时机是什么？哪种训练监督格式能产生最强的适配器？

## 2. 方法论

### 核心思想
通过逐步驱逐文档KV缓存中的键值状态（控制压缩率ρ），比较基模型与加载文档LoRA模型在问答任务上的性能差异（ROUGE-L），从而揭示LoRA的贡献随上下文保留程度的变化规律。

### 关键技术细节

1. **文档专用LoRA训练**：
   - 对每个文档d，将其分割为若干块，从块中生成QA训练数据（默认使用合成QA监督）。
   - 冻结基模型θ₀，仅更新LoRA参数Δθ，优化目标是带损失掩码的下一词预测。
   - LoRA配置：r=16，α=32，dropout=0.1，目标模块为MLP投影；采用DoRA（Weight-Decomposed Low-Rank Adaptation）。

2. **KV缓存压缩**：
   - 使用Compactor算法（基于注意力得分和近似杠杆得分）对文档部分的KV缓存进行驱逐。
   - 压缩率ρ：移除文档键值状态的比例，实验中取值{0, 80%, 90%, 95%, 99%, 99.5%, 100%}。ρ=0保留完整缓存，ρ=100%完全删除文档。

3. **推理阶段控制**：
   - 将推理分为三个阶段：文档预填充（Prefill）、压缩评分（Scoring）、答案解码（Decode）。
   - 定义四种模式：Base（全程无LoRA）、Adapter score + adapter prefill（默认，全程LoRA）、Base KV + adapter decode（基模型做预填充和评分，LoRA仅用于解码）、Base score + adapter prefill（LoRA做预填充和解码，但评分用基模型）。

4. **训练格式控制**：比较四种监督格式：合成QA、原始上下文、上下文续接、上下文+QA。

## 6. 主要结论与发现

1. **LoRA在激进压缩时最有用**：当KV缓存大部分保留时，LoRA几乎不贡献；但随着压缩率增加（尤其>99%），LoRA恢复13-21 ROUGE-L点。LoRA是上下文记忆稀缺时的互补通道，而非替代。
2. **最佳推理策略：基模型预填充，LoRA仅用于解码**：Base KV + adapter decode模式通常最强，说明基模型更适合构建文档KV表示，而LoRA更适合作为解码时的参数记忆。
3. **QA监督远优于其他训练格式**：合成QA监督在无上下文时达到0.19-0.26 ROUGE-L，而原始文本预测仅接近基模型基线。LoRA需要学习“问题→答案”映射，而非简单背诵文档。
4. **MLP是LoRA最有效的目标模块**：MLP-only优于All-linear和Attention-only，支持MLP作为事实记忆存储的观点。
5. **压缩算法鲁棒性**：Compactor和Expected Attention两种算法下，LoRA的增益模式一致，说明结论不依赖于特定压缩器。
