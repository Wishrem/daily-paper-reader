## 摘要
大语言模型（LLMs）的快速普及已将大量推理工作负载转向吞吐量导向的离线场景，此时充分利用GPU算力需要较大的批量大小。然而，现有部署面临结构性矛盾。数据并行（DP）能良好扩展吞吐量，但会复制模型权重，留给键值（KV）缓存的GPU内存有限，从而限制批量大小。模型并行减少每设备权重，但需要细粒度同步，削弱了DP的独立性和调度灵活性。我们提出SiDP，一种用于离线LLM推理的内存高效数据并行范式，它将权重视为DP组内由带宽支持的共享资源。SiDP不将完整模型存储在每个GPU上，而是将权重组织为分布式池：每层由单个GPU拥有，其他副本通过两种互补执行模式按需访问其权重：Weight-as-a-Service（WaS）模式，在大批量场景下通过NVLink将远程权重流式传输到小型缓存中；以及Compute-as-a-Service（CaS）模式，在小批量尾部将激活值发送给所有者。在配备Qwen3-32B、Qwen2.5-72B和Llama-3.1-70B的NVIDIA H20、H200和B200 GPU上评估，SiDP在相同配置下将可用KV容量提升高达1.8倍，并转化为离线工作负载相比基线（vLLM）高达1.5倍的端到端吞吐量提升。

## Abstract
The rapid adoption of large language models (LLMs) has shifted a substantial portion of inference workloads into throughput-oriented offline regimes, where fully utilizing GPU compute requires large batch sizes. However, existing deployments face a structural tension. Data parallelism (DP) scales throughput well but replicates model weights, leaving limited GPU memory for key-value (KV) cache and constraining batch size. Model parallelism reduces per-device weights, but requires fine-grained synchronization that erodes DP's independence and scheduling flexibility. We present SiDP, a memory-efficient data-parallel paradigm for offline LLM inference that treats weights as a bandwidth-backed shared resource inside a DP group. Instead of storing the full model on every GPU, SiDP organizes weights as a distributed pool: each layer is owned by a single GPU, and other replicas access its weights on demand via two complementary execution modes: a Weight-as-a-Service (WaS) mode that streams remote weights over NVLink into a small cache in the large-batch regime, and a Compute-as-a-Service (CaS) mode that ships activations to owners in the small-batch tail. Evaluated on NVIDIA H20, H200, and B200 GPUs with Qwen3-32B, Qwen2.5-72B, and Llama-3.1-70B, SiDP increases usable KV capacity by up to 1.8x under the same configurations, and converts this into up to 1.5x higher end-to-end throughput over baselines (vLLM) for offline workloads.

# 论文总结：SiDP: Memory-Efficient Data Parallelism for Offline LLM Inference

## 1. 核心问题与整体含义（研究动机和背景）
- **背景**：离线LLM推理（如批量评估、文档处理、合成数据生成）以吞吐量（tokens/s）为核心指标，增大批大小（batch size）是提升吞吐量的主要手段，但受限于GPU内存。
- **核心矛盾**：
  - 数据并行（DP）复制完整模型权重，留给KV缓存的剩余内存少，限制了可达的批大小。
  - 模型并行（如TP、PP）虽减少每设备权重，但引入细粒度同步，破坏了DP的独立性和调度灵活性。
- **目标**：在不牺牲DP独立性和调度灵活性的前提下，减少权重冗余，释放更多内存用于KV缓存，从而支撑更大批大小并提升吞吐量。

## 2. 论文提出的方法论
### 核心思想
- 将同一DP组内的FFN权重视为“带宽支持的共享资源”，通过分布式池化存储：每个GPU只拥有部分层的所有权，其他副本按需访问。
- 设计两种互补执行模式，根据批大小动态切换：
  - **Weight-as-a-Service (WaS)**：大批量时，非所有者GPU通过NVLink异步单边读取远程权重到本地小缓存，并在本地执行FFN计算。通过预取和峰值偏移（peak shifting）优化通信。
  - **Compute-as-a-Service (CaS)**：小批量（尾部）时，非所有者将激活值发送给权重所有者，让所有者执行GEMM并返回结果。通过激活融合、跳过虚拟运行等优化。
- **模式切换策略**：由作业协调器（job orchestrator）基于硬件特定的批大小阈值（Bth）在粗粒度边界切换，确保大批量阶段使用WaS，小批量尾部使用CaS。

### 关键技术细节
- **异步缓存管理**：预分配缓存槽，由后台管家线程管理预取和释放，使用CUDA事件跟踪完成。
- **峰值偏移**：通过给每个DP副本施加不同层预取偏移，避免所有副本同时从同一个所有者读取，分散NVLink流量。
- **CaS优化**：
  - GEMM融合：所有者将多个副本的激活拼接后执行单次GEMM，提高利用率。
  - 跳过虚拟运行：空闲副本标记为dummy，不触发任何通信或计算。
- **实现**：作为vLLM插件（约4000行Python），使用CUDA IPC暴露内存，流式异步拷贝。

## 6. 论文的主要结论与发现
- SiDP将可用KV容量提升达1.8倍（尤其在高DP度下），并转化为1.3–1.5×的端到端吞吐量提升。
- WaS在大批量时（B≥128）可完全隐藏权重获取开销；CaS在小批量时（B≤32）明显优于WaS。
- 峰值偏移是DP≥4时的必要条件（提升2.5–3.4×）。
- CaS的优化（异步P2P、GEMM融合、跳过虚拟运行）使尾部性能接近vLLM，整体组合优于WaS单独。
- 长尾场景中，CaS仅占迭代的<10%，但有效避免了尾部拖累。

## 7. 优点
- **创新性**：将权重视为共享资源，同时保留DP独立性和调度灵活性，突破了传统DP/MP二元对立。
- **实用轻量**：作为vLLM插件实现，无需修改框架核心，代码量小，容易集成。
- **性能显著**：跨多个GPU架构与模型一致获得吞吐量提升，尤其适合长上下文、内存受限的离线场景。
- **设计稳健**：通过峰值偏移、缓存管理、模式切换等机制解决实际部署中的通信热点和尾部效应。

## 8. 不足与局限
- **适用范围**：仅针对密集Transformer模型在NVLink连接的单节点内；未处理MoE架构（仅在未来工作提及）。
- **鲁棒性**：权重所有权绑定在特定GPU上，故障域类似于模型并行，不如传统DP容错。
- **多节点扩展**：论文仅设计为单节点内原语，多节点时需复制SiDP组，但未具体评估跨节点通信和调度开销。

- **对短序列场景**：虽然在0.6内存利用率下也有效，但增益较4K时小；当模型较小或DP度低时增益有限。
