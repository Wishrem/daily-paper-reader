## 摘要
随着大语言模型（LLM）服务输入长度的持续增长，KV缓存已成为AI基础设施中的主要瓶颈。它限制了GPU内存容量、服务并发性、缓存复用以及分布式可扩展性。几个重要问题，包括位置无关的KV缓存、前缀KV缓存压缩、热/冷KV缓存分离以及分布式KV缓存管理，都依赖于KV缓存的表示和管理方式。然而，现有的服务系统在很大程度上依赖于单一的KV缓存抽象，即将KV缓存视为同质化的令牌级内存块序列，并在所有注意力头和服务场景中使用相似策略进行管理。我们观察到KV缓存在不同KV头之间具有高度结构化的效用：不同的头表现出不同的功能角色、注意力距离和运行时重要性。因此，并非每个头、令牌范围或服务场景都需要完整的KV缓存。我们提出RedKnot，一种面向LLM服务的头感知KV缓存管理系统。RedKnot通过沿着KV头分解KV缓存来打破传统的单一KV缓存抽象，这些KV头的重要性与有效注意力范围在不同服务场景中显著变化。这种头级分解将KV缓存从单一的张量抽象转变为结构化的内存对象，使RedKnot能够统一支持位置无关的KV复用、前缀KV压缩、热/冷KV分离以及分布式KV放置，同时保持输出保真度并提高资源效率，无需模型重新训练或微调。RedKnot通过将KV缓存从被动的单一运行时构件转变为动态的、感知模型的运行时基板，为可扩展的LLM服务建立了AI基础设施的新基础。

## Abstract
As the input length of large language model (LLM) serving continues to grow, the KV cache has become a dominant bottleneck in AI infrastructure. It limits GPU memory capacity, serving concurrency, cache reuse, and distributed scalability. Several important problems, including position-independent KV cache, prefix KV cache compression, hot/cold KV cache separation, and distributed KV cache management, all depend on how the KV cache is represented and managed. However, existing serving systems largely rely on a monolithic KV cache abstraction, where the KV cache is treated as a homogeneous sequence of token-level memory blocks and managed with similar policies across attention heads and serving scenarios. We observe that KV cache utility is highly structured across KV heads: different heads exhibit different functional roles, attention distances, and runtime importance. Therefore, a full KV cache is not always necessary for every head, token range, or serving scenario.   We present RedKnot, a head-aware KV cache management system for LLM serving. RedKnot breaks the conventional monolithic KV cache abstraction by decomposing the KV cache along KV heads, whose importance and effective attention ranges vary significantly across serving scenarios. This head-level decomposition turns the KV cache from a monolithic tensor abstraction into a structured memory object, enabling RedKnot to uniformly support position-independent KV reuse, prefix KV compression, hot/cold KV separation, and distributed KV placement while preserving output fidelity and improving resource efficiency, without requiring model retraining or fine-tuning. RedKnot establishes a new foundation for AI infrastructure by transforming the KV cache from a monolithic, passive runtime artifact into a dynamic, model-aware runtime substrate for scalable LLM serving.

# RedKnot：面向长上下文LLM服务的头感知KV复用与SegPagedAttention

## 1. 核心问题与整体含义

大语言模型（LLM）服务中，随着输入长度增长，KV缓存成为主要瓶颈：它消耗大量GPU内存、限制并发、阻碍缓存复用和分布式扩展。现有服务系统（如vLLM、SGLang）采用“单一KV缓存抽象”，将KV缓存视为所有注意力头同质的令牌级内存块序列，以相同策略管理。然而，不同注意力头（KV head）在功能角色、注意力距离、运行时重要性上存在显著差异。这种差异未被利用，导致KV缓存利用低效，例如位置无关缓存复用（PIC）中令牌级恢复无法捕获头级别的稀疏性，前缀KV压缩、冷热分离、分布式放置等关键问题均受限于这种同质化管理。论文提出RedKnot，打破传统抽象，将KV缓存按头分解为结构化内存对象，实现统一的、可扩展的LLM服务基础设施。

## 2. 方法论

### 核心思想
将KV缓存的存储、恢复、计算粒度从令牌/层级别对齐到头级别（head-level），利用不同头在功能上的结构性差异（全局头 vs 本地头），实现高效的位置无关KV复用、KV压缩、冷热分离和分布式KV放置，无需模型重训练。

### 关键技术细节
- **头分类（Head-class sparsification）**：离线分析每个（层, 头）对的注意力模式，将其划分为全局头（global head，12–15%，需要全序列注意力）和本地头（local head，85–88%，仅需窗口内令牌）。该分类在不同请求间稳定。
- **RoPE位置对齐**：复用缓存时，通过RoPE旋转矩阵将缓存的键（K）从离线位置旋转到在线位置，消除位置不匹配。
- **弹性稀疏恢复（Elastic Sparsity）**：层级恢复策略。浅层使用本地注意力恢复+全FFN（保守策略）；深层执行头感知注意力恢复（全局头全范围重计算，本地头仅修复其可见窗口内的令牌） + 部分稀疏FFN（仅对注意力信号重要的令牌执行FFN）。
- **SegPagedAttention**：一种新的KV缓存物理布局和内核。将KV缓存按（层, 头, 段）组织为独立页面，每个头拥有自己的页面表，支持变长注意力。内核使用融合变长FlashAttention避免构建注意力掩码，从而消除掩码惩罚（4.9–7.6×），使算法节约的字节真正转化为计算加速。
- **稀疏FFN**：基于恢复后的注意力信号选取top-k重要令牌进行评估，其余令牌通过残差路径跳过FFN，独立于上下文长度，加速短上下文agent负载。

### 算法流程（文字描述）
1. 离线：对每个模型运行推理，构建（层, 头）的全局/本地哈希映射。
2. 以头粒度存储KV缓存，建立每个头的段到虚拟页面的映射。
3. 在线请求到达时，完全重计算全局头的KV；本地头大部分复用，仅修复小部分（本地窗口+注意sink令牌）。
4. 每层注意力恢复后，对重要令牌执行稀疏FFN，其他令牌走残差路径。

## 6. 主要结论与发现

1. **RedKnot在质量和TTFT上优于令牌级PIC基线**：在Llama-70B 64K上，TTFT加速比最大，同时EM和F1高于或等于基线。
2. **预填充FLOPs大幅降低**：最高减少79.5%（如Llama-70B 64K从21.4 PFLOPs降至2.64 PFLOPs），原因是头级恢复避免了令牌级联合效应，且稀疏FFN进一步减少计算。
3. **SegPagedAttention消除掩码惩罚**：相比Dense+mask，解码加速2.5-21.0×，预填充加速6.3-23.3×，且速度提升随上下文长度单调增长。
4. **系统级收益**：KV传输字节减少4.3-6.3×，突发吞吐量提升15-43%，并发容量提升4.7-7.8×（在SegPagedAttention物理实现下）。
5. **关键洞察**：对齐恢复、存储、计算粒度与头级稀疏结构是突破瓶颈的关键；头级分解应成为长上下文服务引擎的基本抽象。
