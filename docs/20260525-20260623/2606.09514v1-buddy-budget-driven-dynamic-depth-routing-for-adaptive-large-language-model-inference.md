## 摘要
大型语言模型（LLM）因其深度和参数规模而产生高推理成本。深度剪枝可通过跳过冗余Transformer块来减少延迟，但现有方法（i）在用户特定计算预算下提供的控制有限，且（ii）通常固定路由路径，无法在解码过程中随上下文增长而调整。我们提出Buddy，一种预算驱动的动态深度路由框架。Buddy使用轻量级决策模块根据输入对中间层进行评分，并确定性地执行top-k层以满足给定预算。为支持解码时自适应，Buddy复用第一层KV缓存作为低开销的全局上下文源，并在每次路由决策前将其与最新的token表示合并。当未提供显式预算时，可选的预算预测器会估计输入相关的计算级别以平衡质量与效率。在Llama系列和Qwen模型上的实验表明，Buddy与强静态剪枝基线相比具有竞争力，且通常能改善准确率-计算量权衡，同时独特地支持在单个训练模型内的严格预算控制、解码时重路由和多种预算。

## Abstract
Large language models (LLMs) incur high inference cost due to their depth and parameter scale. Depth pruning can reduce latency by skipping redundant Transformer blocks, but existing methods (i) provide limited control under user-specific compute budgets and (ii) typically fix the routing path, failing to adapt as the context grows during decoding. We propose Buddy, a budget-driven dynamic depth routing framework. Buddy uses a lightweight Decision Module to score intermediate layers conditioned on the input and deterministically executes the top-k layers to satisfy a given budget. To support decode-time adaptation, Buddy reuses the first-layer KV cache as a low-overhead global context source and pools it together with the newest token representation before each routing decision. When no explicit budget is provided, an optional Budget Predictor estimates an input-dependent compute level to balance quality and efficiency. Experiments on Llama-family and Qwen models show that Buddy is competitive with strong static pruning baselines and often improves the accuracy-compute trade-off, while uniquely supporting strict budget control, decode-time rerouting, and multiple budgets within a single trained model.

### 2. 论文提出的方法论
- **核心思想**：  
  一个轻量级 **决策模块**（Decision Module）根据当前上下文对中间层评分，然后确定性地选择 top-k 层执行，其中 k 由用户预算（执行比例 b）决定。为支持解码时重路由，**复用第一层 KV 缓存** 作为低开销全局上下文，与最新 token 表示拼接后输入决策模块。可选 **预算预测器**（Budget Predictor）在用户未指定预算时自动推断每输入的合适计算量。

- **关键技术细节**：
  - **层跳过公式**：隐藏状态更新为 \(H_\ell = F_\ell(H_{\ell-1}) + H_{\ell-1}\) 若 \(M_\ell=1\)（执行），否则 \(H_\ell = H_{\ell-1}\)（跳过）。
  - **决策模块**：  
    - 对中间层（排除首尾层）输出分数 \(s\)，可选融合先验（经归一化的离线重要性指标，如 ΔPPL、Taylor、余弦相似度等）。  
    - 使用 Softmax + Top-k 生成二进制掩码 M，k = round(b·L) - 2。  
    - 训练时使用直通估计器（STE）使梯度流过离散选择。
  - **KV 感知规划器**：在解码每一步，从第一层 KV 缓存中取 Key/Value 状态（消融发现 Value 状态最优），与当前 token 特征拼接后输入决策模块，从而更新路由路径。
  - **预算预测器**：当无用户预算时，使用 GRPO 训练一个轻量策略网络，输出离散层数（1~Lmid），奖励函数平衡 CE 损失与计算开销。

- **算法流程（文字描述）**：  
  1. 输入 prompt x 和可选预算 b。  
  2. 从第一层 KV 缓存提取全局上下文 G。  
  3. 决策模块基于 G 和 b 输出层掩码 M（确定 top-k）。  
  4. 执行时只通过选中的中间层（首尾层始终执行）。  
  5. 解码每一步，更新第一层 KV 缓存并重新执行步骤 2-4。  
  6. 若无预算，先由预算预测器估计 b。

### 7. 优点
- **预算驱动的确定性控制**：用户可指定精确的执行比例，训练后无需额外校准。  
- **解码时动态重路由**：利用已有的 KV 缓存，几乎无额外开销。  
- **单模型多预算**：无需为每个稀疏度训练独立模型，降低部署成本。  
- **先验知识融合**：可选的离线重要性指标加速训练并提升效果。  
- **轻量决策模块**：理论 FLOPs 仅占主干网络的 0.003%~0.005%，实际延迟占比约 5.67%。  
- **全面消融实验**：验证了各组件（特征、先验、层数、系数）的影响。  
- **开源倾向**：论文提供了 Hugging Face 链接和训练细节（附录），可复现。

### 8. 不足与局限
- **VRAM 未减少**：所有 Transformer 块仍常驻内存，仅跳过计算不释放显存。  
- **批处理低效**：不同序列路径不同导致 KV 缓存未命中，当前使用零填充处理，未来需更高效策略。  
- **仅验证 7B-13B 规模**：未在更大模型（如 70B）上评估可扩展性。  
- **极高稀疏度性能下降**：移除 50% 以上层时准确率下降明显（如 Llama3-8B 仅 71.3% 保留）。  
- **预算预测器需额外训练**：GRPO 阶段的超参数（λperf, λcost）可能需针对不同任务调整。  
- **长序列场景未充分讨论**：上下文增长至数千 token 时，首层 KV 缓存能否有效代表全局有待验证。  
- **未与最新稀疏架构（如 MoD、MoE）对比**：基线仅包含传统深度剪枝方法。
