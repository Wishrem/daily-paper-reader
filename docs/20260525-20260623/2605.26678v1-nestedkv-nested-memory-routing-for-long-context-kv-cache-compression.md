## 摘要
长上下文语言模型受到键值（KV）缓存内存占用的限制。现有的免训练KV压缩方法通常基于单一重要性信号（注意力、时效性、逐层分配或键的独特性）对令牌进行排序，当有用的上下文具有全局独特性、局部片段性或即时相关性时，这些方法会变得脆弱。我们提出NestedKV，一种仅键的KV缓存压缩方法，受嵌套学习中的连续内存系统启发。NestedKV维护全局、块级别和滑动窗口键锚点，通过多时间尺度余弦异常对令牌进行评分，并将得到的排名与使用头部自适应混合和惊异门控令牌路由的免训练外部学习器相结合。该分数与自适应的每头预算配对，无需训练或修改LLM。在RULER（4k–32k）、LooGLE、LongBench、LongBench-E、InfiniteBench和MMLU-Pro上，使用Qwen3和Llama-3.2模型，当保留的缓存较小时，NestedKV表现最强。在Qwen3-4B上，在压缩率r=0.75时，它在RULER上比KeyDiff提升高达19.10分，在LongBench上提升19.29分；在r=0.95时，它在LongBench上保留37.32分，而KeyDiff为17.55分。

## Abstract
Long-context language models are limited by the memory footprint of the key-value (KV) cache. Existing training-free KV compression methods usually rank tokens by one importance signal -- attention, recency, layer-wise allocation, or key distinctiveness -- which becomes brittle when useful context is globally distinctive, locally episodic, or immediately relevant. We introduce NestedKV, a key-only KV cache compression method inspired by the Continuum Memory System in Nested Learning. NestedKV maintains global, block-level, and sliding-window key anchors, scores tokens by multi-time-scale cosine anomaly, and combines the resulting rankings with a training-free outer learner using head-adaptive mixing and surprise-gated token routing. The score is paired with adaptive per-head budgets and requires no training or LLM modification. Across RULER (4k--32k), LooGLE, LongBench, LongBench-E, InfiniteBench, and MMLU-Pro on Qwen3 and Llama-3.2 models, NestedKV is strongest when the retained cache is small. On Qwen3-4B, it improves over KeyDiff by up to 19.10 points on RULER and 19.29 on LongBench at $r=0.75$; at $r=0.95$, it retains 37.32 on LongBench versus 17.55 for KeyDiff.

## 1. 论文的核心问题与整体含义（研究动机和背景）

- **核心问题**：长上下文语言模型（LLM）的键值（KV）缓存随序列长度线性增长，成为部署瓶颈。现有免训练的KV压缩方法通常只依赖一种重要性信号（如注意力分数、时效性、逐层分配或键的独特性）来排序和丢弃令牌，但在上下文的全局独特性、局部片段性或即时相关性共存时，单一信号排序会变得脆弱，导致错误地丢弃关键信息。
- **研究动机**：作者发现，即使注意力机制本身也会将质量集中在前缀注意力汇和提示尾部，而实际包含答案的令牌往往处于低注意力区域。因此，需要跳出注意力空间，从键流中提取更丰富的多尺度记忆信号。
- **整体含义**：将KV缓存压缩重新定义为连续内存系统中的异常检测问题，通过多个时间尺度的键统计量（稳定、片段、当前）来评估令牌的“不可预测性”，从而在压缩时保留更有用的上下文。

## 2. 论文提出的方法论：核心思想、关键技术细节

### 核心思想
- **连续内存系统（Continuum Memory System）**：维护三种时间尺度的内存锚点，分别捕捉文档级稳定的全局模式、块级局部片段模式、滑动窗口级即时流模式。
- **多尺度余弦异常评分**：对每个缓存键，计算其与三种记忆锚点的余弦相似度，取负作为异常分数（高异常意味着该键无法被当前记忆状态很好预测，应保留）。
- **免训练外部学习器**：通过头自适应混合（对各注意力头，根据各尺度的区分能力动态加权）和惊异门控路由（当三个尺度评分不一致时，用最大单尺度评分代替混合评分），组合内层学习器的排名结果。
- **自适应每头预算分配**：让各注意力头在全局竞争预算，允许不同头专注于不同时间尺度的残差结构。

### 关键技术细节
- **稳定记忆**：整段预填充上下文的键均值 $\mu_s$。
- **片段记忆**：令牌所在的大块（块大小 clip(⌊N/32⌋, 128, 256)）内的键均值 $\mu_e(i)$。
- **当前记忆**：最近W=64个键的滑动窗口均值 $\mu_c(i)$。
- **三个尺度异常分数**：$a_s(i) = -\cos(\hat{k}_i, \mu_s)$，$a_e(i) = -\cos(\hat{k}_i, \mu_e(i))$，$a_c(i) = -\cos(\hat{k}_i, \mu_c(i))$。经过min-max归一化得到 $\tilde{a}_s, \tilde{a}_e, \tilde{a}_c$。
- **头自适应混合权重**：对每个头，计算每个尺度区分其前10%与后10%令牌的差距 $\Delta_k$，通过带固定先验的 softmax 得到权重 $w_k$（先验 (0.4,0.4,0.2)）。混合分数 $a_{\text{blend}}(i) = w_s \tilde{a}_s(i) + w_e \tilde{a}_e(i) + w_c \tilde{a}_c(i)$。
- **惊异门控路由**：定义惊异 $s(i) = \text{std}(\tilde{a}_s, \tilde{a}_e, \tilde{a}_c)$。若惊异低（三尺度一致），使用混合分数；若惊异高（三尺度冲突），使用最大单尺度分数 $a_{\text{win}}(i) = \max(\tilde{a}_s, \tilde{a}_e, \tilde{a}_c)$。通过 sigmoid 门控 $\alpha(i) = \sigma(\kappa (s(i) - \tau))$ 进行插值。
- **压缩操作**：最终分数 $a^*(i)$ 取Top-B保留，前4个令牌固定保留（注意力汇）。
- **头间预算竞争**：每层所有头令牌对按分数排序，全局选取B个最高分数对，同时确保每头至少保留最小数量。

### 伪算法流程（文字说明）
1. 预填充后，对每层每头计算三个记忆锚点（稳定、片段、当前）。
2. 对每个缓存键计算三个余弦异常分数，归一化。
3. 对每个头，根据每个尺度的区分能力计算混合权重，得到混合分数。
4. 对每个令牌，计算三个异常分数的标准差作为惊异，根据惊异门控混合与最大单尺度分数得到最终分数。
5. 在层内对头-令牌对进行全局竞争，分配各头的保留预算。
6. 每头根据预算保留得分最高的令牌。

## 6. 论文的主要结论与发现

- **主要结论**：NestedKV 在低保留率（高压缩）和长上下文场景下显著优于现有免训练压缩方法。当压缩比增大时，其性能下降最平缓，体现了多尺度记忆的优势。
- **关键数据**：

## 7. 优点

1. **免训练**：完全不需要任何额外训练或模型微调，可直接用于冻结的 LLM。
2. **多尺度记忆**：不是单一重要性信号，而是三个互补的时间尺度，更能捕获不同性质的上下文信息。
3. **头自适应混合**：不同注意力头可以专注于不同记忆尺度，更灵活。
4. **惊异门控路由**：处理内存冲突时自动加强最强信号，避免平均导致的盲点。
5. **自适应预算分配**：允许头间根据残差大小竞争预算，而非固定均匀分配。
6. **兼容标准注意力**：只决定保留哪些键值，保留后 KV 不变，可直接使用优化后的注意力内核。
7. **鲁棒性强**：在极低保留率（r=0.95）下仍保持可观性能，远超基线。

## 8. 不足与局限

1. **对代码补全等任务可能较弱**：连续记忆假设“冗余”可由统计平均值捕获，但代码中局部重复可能是关键模式（如循环体），此方法可能会错误删除。
2. **仅预填充压缩**：NestedKV 仅在预填充后进行一次压缩，不适用于生成过程中自适应更新（虽然论文提到未在生成长序列时重新计算权重，留作未来工作）。
3. **实验模型规模有限**：最大为 8B 参数（Qwen3-8B），未在更大模型（如 70B+）上验证。在更大模型中多尺度行为可能不同。
4. **缺乏统计显著性检验**：仅报告平均分数，未提供多次运行的方差或置信区间，无法判断差异是否统计显著。
5. **超参数敏感性实验范围较窄**：虽然结果显示稳健，但仅在默认值附近稍作变化，未在更大范围内测试。
6. **缺乏端到端真实世界下游任务评估**：如长文档摘要、多轮对话等，基准多为合成或标准问答，可能无法完全反映实际应用中的行为。
7. **未讨论训练免方法相比于学习/查询感知方法的局限**：论文强调免训练，但可能牺牲了在困难场景下的上限。
