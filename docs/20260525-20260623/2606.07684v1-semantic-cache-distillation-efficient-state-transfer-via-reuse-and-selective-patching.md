## 摘要
分离式服务缓解了大语言模型推理中的内存瓶颈，但造成了严重的通信瓶颈：传输高维键值缓存通常占据首令牌时间的主导地位。此外，在异构模型（如基础模型与微调变体）间重用缓存会导致语义错位，且该错位随层数累积，降低生成质量。我们提出语义缓存蒸馏，一种损失约束框架，用紧凑语义编码替代原始键值传输。该框架通过两种机制应对挑战：（1）重用，从低秩子空间重建大部分层以最小化传输成本；（2）修补，在稀疏过渡层预测归一化输入以截断误差传播。实验表明，在带宽受限场景下，语义缓存蒸馏相比预言式消费者预填充实现了高达2.65倍的首令牌时间加速，并在质量-延迟帕累托前沿上优于量化与选择性重计算基线，同时将生成质量控制在预言式F1分数的5%以内。

## Abstract
Disaggregated serving alleviates memory bottlenecks in Large Language Model (LLM) inference but creates a severe communication bottleneck: transmitting high-dimensional Key-Value (KV) caches often dominates time-to-first-token (TTFT). Moreover, reusing caches across heterogeneous models (e.g., base and fine-tuned variants) causes semantic misalignment that accumulates over layers, degrading generation quality. We propose Semantic Cache Distillation (SCD), a loss-constrained framework that replaces raw KV transmission with compact semantic codes. SCD addresses these challenges via two mechanisms: (1) Reuse, which reconstructs most layers from low-rank subspaces to minimize transfer cost, and (2) Patch, which predicts normalized inputs at sparse transition layers to truncate error propagation. Empirically, SCD delivers up to 2.65 $\times$ TTFT speedup over the oracle consumer prefill and dominates quantization and selective recomputation baselines on the quality--latency Pareto frontier in bandwidth-constrained regimes, while keeping generation quality within 5\% F1 of the oracle.

## 1. 核心问题与整体含义（研究动机和背景）

- **问题**：分离式大语言模型推理中，预填充阶段与解码阶段分跨不同设备，虽缓解了内存瓶颈，但高维键值缓存的网络传输成为新的主要瓶颈，首令牌时间严重受限于带宽。此外，当生产者和消费者模型权重不同（如基础模型与微调变体）时，直接重用 KV 缓存会导致**语义漂移**，该漂移随层数累积，显著降低生成质量。
- **背景**：现有方法（如量化、稀疏化、低秩压缩）主要面向同模型场景，未处理异构模型间的特征空间差异；DroidSpeak 等选择性重计算方法强制在重用与重计算间刚性权衡，无法同时优化延迟与质量。
- **整体含义**：论文旨在设计一个带宽高效、语义对齐的状态传输框架，使得权重不匹配但共享架构的模型对（如基础模型→微调专家）能够在分离式推理中实现低延迟、高质量的状态复用。

## 2. 方法论：核心思想、关键技术细节与算法流程

- **核心思想**：用紧凑语义编码替代原始 KV 传输，通过**Reuse**（低秩重构）覆盖大部分层以最小化传输量，通过**Patch**（稀疏过渡层语义校正）截断误差传播，保持生成质量。
- **关键技术细节**：
  - **Reuse 机制**：
    - 离线阶段：从校准数据集中收集生产者与消费者成对的 KV 张量，对每个层独立进行联合低秩分解（截断奇异值分解），得到共享潜在编码和各自解码器；利用脊回归学习生产者编码器。
    - 在线阶段：生产者对 Key 去旋转位置编码后，通过编码器生成低维编码 \( Z^K, Z^V \)；消费者通过解码器恢复本地空间的 KV 对，再重新应用旋转位置编码。
  - **Patch 机制**：
    - 在少量“过渡层”（通过离线贪心选择）上，传输生产者归一化的预注意力隐藏状态的低秩编码。
    - 消费者使用一个轻量多层感知机构成的对齐器 \( g_\theta \) 将编码映射到本地空间，然后利用消费者自身的投影权重重新生成该层的 KV 对，从而“重置”误差累积。
  - **理论分析**：证明无 Patch 时误差随层数乘积放大；Patch 作为谱截断算子将误差限制在局部段内，确保负对数似然差距有界。
- **算法流程**（文字描述）：
  1. 离线校准：对固定模型对，使用少量前缀运行全预填充，记录成对 KV 和归一化隐藏状态；拟合 Reuse 的低秩映射和 Patch 的对齐器；按效率指标选择 Patch 层集合。
  2. 在线推理：
     - 生产者预填充后，对 Reuse 层编码 \( Z^K, Z^V \)，对 Patch 层编码 \( Z^H \)，传输所有编码。
     - 消费者对 Reuse 层解码得到 KV 缓存；对 Patch 层用对齐器映射得到隐藏状态，再用本地权重生成 KV 缓存。
     - 组装混合缓存后直接进入解码循环。

## 6. 主要结论与发现

- **主要结论**：SCD 在带宽受限的分离式推理中实现了**首令牌时间速度提升最高达 2.65 倍**（相对于 Oracle 全预填充），同时生成质量 F1 分数与 Oracle 差距在 5% 以内（7B 模型 0.785 vs 0.807；32B 模型 0.847 vs 0.876）。
- **关键发现**：
  - 传统量化（4-bit）无法解决跨模型语义差距，导致质量崩溃（F1仅 0.13）。
  - Reuse-only 可压缩传输但仍有较大质量损失（F1 0.64）；加入稀疏 Patch 后恢复至接近 Oracle。
  - 误差传播理论得到实验验证：无 Patch 时每层误差累积放大，Patch 在过渡层有效截断。
  - 少量关键层的语义纠正即可大幅提升整体生成质量，验证了“重用为主、修补为辅”策略的有效性。
