## 摘要
推理模型通过扩展思维链提高准确性，但其长输出造成了内存和计算瓶颈。KV缓存淘汰方法通过从缓存中淘汰不重要的键值对来降低这一成本，但相比保留完整KV缓存的基于选择的稀疏注意力替代方案，它们往往导致更差的准确性。我们识别了影响KV缓存淘汰准确性的关键因素。首先，一小部分值状态具有异常大的幅度，淘汰它们会导致模型进入重复推理循环的灾难性失败。其次，在淘汰过程中引入随机性通过增加缓存多样性提高了准确性。基于这些发现，我们提出了价值感知随机KV缓存淘汰（VaSE），这是一种无需训练的配方，保护大幅值状态并促进多样化的淘汰决策。在六个推理任务上，使用VaSE进行4倍KV缓存压缩的Qwen3模型在同一稀疏度下获得了比最先进选择方法更高的平均准确率，同时比最强的淘汰方法高出4%以上。总体而言，VaSE弥合了效率与准确性之间的差距，支持FlashAttention2，并为推理模型实现了静态内存占用。

## Abstract
Reasoning models improve accuracy through extended chains of thought, but their long outputs create a memory and compute bottleneck. KV cache eviction methods reduce this cost by evicting unimportant key-value pairs from the cache, yet they often yield worse accuracy than selection-based sparse attention alternatives, which keep the full KV cache. We identify key factors crucial to KV cache eviction accuracy. First, a small fraction of value states have abnormally large magnitudes, and evicting them causes catastrophic failure where models enter repetitive reasoning loops. Second, introducing stochasticity during eviction improves accuracy by increasing cache diversity. Based on these findings, we propose Value-aware Stochastic KV Cache Eviction (VaSE), a training-free recipe that protects large-magnitude value states and promotes diverse eviction decisions. Across six reasoning tasks, Qwen3 models using VaSE with 4x KV cache compression yield higher average accuracies than SOTA selection method at the same sparsity, while outperforming the strongest eviction method by more than 4%. Overall, VaSE bridges the gap between efficiency and accuracy, supporting FlashAttention2 and enabling a static memory footprint for reasoning models.

### 论文结构化总结：Value-Aware Stochastic KV Cache Eviction for Reasoning Models

#### 1. 核心问题与整体含义 (研究动机和背景)

- **核心问题**：推理模型（如 DeepSeek-R1, QwQ）通过生成极长的思维链来提升准确性，但这会导致 **KV Cache** 规模急剧膨胀，造成严重的**内存瓶颈**和**计算开销**。现有的 KV Cache 压缩方法存在权衡：
    - **选择型方法（Selection-based）**：保留所有 KV 对，但仅选择性地计算注意力。该方法计算效率高，但内存消耗仍随序列长度线性增长。
    - **淘汰型方法（Eviction-based）**：永久丢弃不重要的 KV 对，能将内存占用固定（静态内存），但会丢失信息，导致准确性显著下降，尤其是在需要长上下文推理的模型上。
- **论文含义**：论文旨在弥合淘汰型方法与选择型方法之间在准确性上的差距，找到一种既能实现高压缩比、静态内存占用，又能保持高准确率的 KV Cache 淘汰策略。

#### 2. 方法论：核心思想、关键技术细节

- **核心思想**：作者识别出两个提高淘汰方法准确性的关键因素：
    1.  **保护大幅值（Value State）**：一小部分 **值向量（value vectors）** 的幅度（用 Range 衡量）异常大，它们是维持模型正确推理的关键。淘汰它们会导致模型陷入**重复推理循环**，无法给出最终答案。
    2.  **引入随机性（Stochasticity）**：在淘汰决策中引入随机抽样，可以增加缓存中 KV Cache 的**多样性**，使得保留的 KV 对能更全面地代表完整的上下文。

- **技术实现（VaSE框架）**：基于以上两个原则，作者提出了 **VaSE (Value-aware Stochastic KV Cache Eviction)**，这是一个无需训练的（training-free）优化配方。
    - **变体一：VaSE-AttnV**
        - **基础**：基于 SnapKV 方法。
        - **值感知**：从待淘汰的候选集合中，预先保留一定数量（Nv）的、具有最大 `Range(vi)` 的值向量，使其“免疫”于淘汰。
        - **随机性**：在剩余预算中，不再使用贪心的 top-k 选择，而是**根据注意力分数（`¯αi`）进行加权随机抽样**，决定哪些 KV 对被保留。
    - **变体二：VaSE-DKV**
        - **基础**：基于 CurDKV 方法。
        - **值感知**：其评分函数本身通过 `ℓ(V)i(G)` 即值向量的投影范数，天然地倾向于大幅值的向量。
        - **随机性**：每次进行淘汰决策时，**重新随机采样一个高斯投影矩阵 Gt**，用于计算评分，从而打破 CurDKV 中固定投影矩阵导致的“永久性”淘汰，提升了缓存多样性。

- **核心公式/算法流程文字说明**：
    - **评分函数改进核心**：将值幅值度量（`Range(v)`）和随机性（`sampling/random projection`）整合到传统的注意力分数（`¯αi`）或矩阵分解分数（`ℓ(K)i * ℓ(V)i`）中。
    - **算法流程（以 VaSE-AttnV 为例）**：
        1.  **周期性淘汰**：每生成 B 个 Token 触发一次淘汰。
        2.  **值保护**：从当前缓存（不含 Buffer 中的最近 B 个 Token）中，挑选出 `Range(v)` 最大的 Nv 个 KV 对，无条件保留。
        3.  **随机淘汰**：从剩余的候选 KV 对中，按其注意力分数 `¯αi` 作为权重，进行随机采样，填满剩余的预算。
        4.  **更新缓存**：将 Buffer 中的 B 个最新 Token 加入缓存，丢弃被淘汰的 KV 对，缓存恢复到预设预算 K。
