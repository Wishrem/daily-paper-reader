## 摘要
长上下文大语言模型推理受到巨大的键值（KV）缓存的严重瓶颈限制，而现有的稀疏注意力方法往往依赖于静态固定预算（Top-k）检索，或使用计算成本高且有偏的代理分数。为了解决这些局限性，我们提出了RaBitQCache，一种新颖的稀疏注意力框架，它利用随机旋转二值量化和高吞吐量的二值-INT4算术来高效估计注意力权重。我们的代理分数作为无偏估计量，具有经过验证的误差界，能够实现自适应的Top-p检索，根据实际的注意力稀疏性动态调整令牌预算。我们进一步实现了一个具有异步流水线和延迟更新的硬件感知系统，以掩盖开销。评估表明，与最先进的基线相比，RaBitQCache在保持生成质量的同时显著加速了推理并减少了内存I/O。代码可在 https://github.com/Sakuraaa0/RaBitQCache.git 获取。

## Abstract
Long-context Large Language Model inference is severely bottlenecked by the massive Key-Value (KV) cache, yet existing sparse attention methods often suffer from static fixed-budget (Top-k) retrieval or rely on proxy scores that are computationally expensive and biased. To address these limitations, we propose RaBitQCache, a novel sparse attention framework that utilizes randomized rotated binary quantization and high-throughput binary-INT4 arithmetic to efficiently estimate attention weights. Our proxy score serves as an unbiased estimator with a proven error bound, enabling adaptive Top-p retrieval that dynamically adjusts the token budget based on actual attention sparsity. We further implement a hardware-aware system with asynchronous pipelining and lazy updates to mask overhead. Evaluations demonstrate that RaBitQCache significantly accelerates inference and reduces memory I/O while preserving generation quality compared to state-of-the-art baselines. Code is available at https://github.com/Sakuraaa0/RaBitQCache.git.

## 1. 核心问题与整体含义（研究动机和背景）

- **核心问题**：长上下文大语言模型（LLM）推理的巨大瓶颈在于**Key-Value (KV)缓存**的显存消耗和内存I/O开销，导致生成延迟随序列长度二次增长。
- **现有方法局限**：
    - 静态固定预算（Top-k）稀疏注意力缺乏对实际注意力稀疏性的适应性，导致要么计算浪费、要么精度下降。
    - 动态方法（如Quest、Double Sparse）使用代理分数估计注意力权重，但代理分数计算昂贵或有偏，且缺乏理论误差界，因此无法支持自适应Top-p检索。
- **研究目标**：提出一种高效、精确、通用的稀疏注意力框架，满足三个核心要求：① 代理分数计算开销低；② 代理分数精度有理论保证；③ 能够自适应调整检索预算。

## 2. 方法论

### 核心思想
- 利用**Johnson-Lindenstrauss (JL)引理**将高维向量映射到低维空间，保持内积近似。
- 将Key向量随机旋转后二值量化到超立方体顶点（1-bit），Query向量量化到INT4，通过高效的二进制-INT4点积计算得到无偏内积估计，从而构造低开销、有理论误差界的代理分数。
- 利用该无偏估计实现**自适应Top-p检索**，动态选择注意力分数累计质量达到p的最小token子集，替代固定预算Top-k。

### 关键技术细节
1. **预填充阶段（索引构建）**（算法1）：
    - 对Key向量进行中心化（减去中心 $c_k$）和归一化得到 $k_c$。
    - 应用随机正交旋转矩阵 $P$：$K' = (K - c_k)P$。
    - 提取旋转后向量的符号位作为1-bit二进制索引 $\bar{C}_b$（$D$位）。
    - 计算每个token的矫正因子 $\alpha = \langle \bar{c}, P^T k_c \rangle$，其中 $\bar{c}$ 是从二进制索引恢复的码本向量。

2. **解码阶段（在线检索）**（算法2）：
    - 对当前query $q_t$ 同样中心化、归一化、旋转得到 $q'$。
    - 将 $q'$ 均匀量化到4-bit整数向量 $\bar{q}_u$。
    - 执行 **INT4 × 二进制 GEMV** 计算 $\langle \bar{C}_b, \bar{q}_u \rangle$，通过公式重建 $\langle \bar{c}, q' \rangle$，再除以 $\alpha$ 得到归一化内积 $\langle q_c, k_c \rangle$。
    - 结合式(2)恢复原始注意力分数，利用**Top-p**选择阈值 $\theta$ 筛选重要token。
    - KV缓存检索后，仅对选中的子集（加上局部滑动窗口）执行完整注意力计算。

3. **系统级优化**：
    - **异步流水线**：预填充阶段在次要流中异步构建索引，与主流$O(L^2)$密集注意力重叠，隐藏索引开销。
    - **延迟更新**：解码阶段新token先暂存在局部窗口缓存中，批量量化以降低内核启动开销。

4. **高效内核实现**：
    - Top-p操作采用三元搜索（三次线性扫描）代替排序，避免$O(L \log L)$瓶颈。
    - INT4×二进制 GEMV 内核采用**打包二进制表示**（32bits打包）、**共享内存分块**缓存query、**向量化位提取**与循环展开。

### 理论保证
- **定理A.1**：代理分数 $\frac{\langle \bar{k}_c, q_c \rangle}{\langle \bar{k}_c, k_c \rangle}$ 是 $\langle k_c, q_c \rangle$ 的无偏估计，误差 $O(1/\sqrt{D})$ 高概率成立。
- **定理A.2**：将 $q'$ 量化到 $B_q = \Omega(\log\log D)$ bits（实际取4bits），引入的量化误差远小于 $O(1/\sqrt{D})$，可忽略。
