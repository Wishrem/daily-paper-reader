## 摘要
随着大语言模型（LLM）推理从纯文本范式演进到多模态范式，推理系统面临三大挑战：（1）多模态工作流的灵活编排，异构计算单元表现出复杂的依赖关系和并发控制；（2）跨进程和节点的高效中间数据传输，张量在异构角色间高速流动；（3）跨角色的KV缓存和模型权重高效共享，以消除冗余的GPU内存。现有解决方案独立部署LLM和扩散模型，缺乏多模态管线的系统级抽象；这导致编排逻辑分散，传输路径与特定模型紧密耦合，并且集成新模型的成本很高。为应对这些挑战，我们提出Omni-Flow，一个通过三层抽象实现多模态推理的分布式调度框架。控制流层通过Python DSL定义工作流，将异构单元编排成统一的数据流图，支持静态DAG和动态路由，并内置服务发现和多种负载均衡策略。数据流层提供超越预填充/解码分离的分布式KV缓存抽象，统一分配并支持跨三级分页存储层次（GPU/CPU/SSD）的直接跨角色传输，通过零拷贝、低延迟通道实现。计算流层支持复杂的多模态前缀匹配以实现多轮对话中的KV复用，并通过统一的SGLang接口接管KV缓存和采样逻辑，使扩散模型在统一并行语义下直接复用LLM前向路径。我们证明Omni-Flow能够以一致的编程模型支持多种异构场景，包括全模态对话（LongCat-Next）和复杂图像生成管线（HunyuanImage-3）。

## Abstract
As large language model (LLM) inference evolves from text-only to multimodal paradigms, inference systems face three challenges: (1) flexible orchestration of multimodal workflows, where heterogeneous computing units exhibit complex dependencies and concurrent control; (2) efficient transmission of massive intermediate data across processes and nodes, with tensors flowing at high speed among heterogeneous roles; and (3) efficient sharing of KV caches and model weights across roles to eliminate redundant GPU memory. Existing solutions deploy LLMs and diffusion models independently, lacking a system-level abstraction for multimodal pipelines; this scatters orchestration logic, tightly couples transmission paths to specific models, and incurs high cost to integrate new models. To address these challenges, we present Omni-Flow, a distributed scheduling framework for multimodal inference through a three-layer abstraction. The Control Flow layer defines workflows via a Python DSL, orchestrating heterogeneous units into a unified dataflow graph that supports static DAGs and dynamic routing, with built-in service discovery and diverse load-balancing strategies. The Data Flow layer provides a distributed KV cache abstraction beyond prefill/decode separation, unifying allocation and enabling direct cross-role transmission across a three-tier paged storage hierarchy (GPU/CPU/SSD) over zero-copy, low-latency channels. The Compute Flow layer supports complex multimodal prefix matching for KV reuse across multi-turn dialogues, and takes over KV cache and sampling logic via a unified SGLang interface, letting diffusion models directly reuse the LLM forward path under unified parallel semantics. We demonstrate that Omni-Flow supports diverse heterogeneous scenarios with a consistent programming model, including omni-modal dialogue (LongCat-Next) and complex image generation pipelines (HunyuanImage-3).

### 2. 论文提出的方法论：核心思想、关键技术细节

#### 2.1 控制流（Control Flow）层
- **声明式图 DSL**：用户通过 Python DSL 定义工作流为有向图（支持环），节点为异构计算角色（如 LLM、Diffusion），边为数据传输通道。
- **三种核心依赖解决**：
  - **菱形依赖**（多路径汇聚）：使用 **Bucket 模型**，通过 (gc_rid, owner_node, or_group, activation_id) 四元组唯一标识汇聚点；activation_id 由共同分叉源的帧序号哈希得到，确保同源数据落入同一 Bucket，不同帧落入不同 Bucket。
  - **序列依赖**（有序输出可能乱序到达）：在 **SessionLoadBalance** 节点维护期望序列号，仅允许匹配序列号的帧通过，待该序列号所有 OR 组完成后才推进。
  - **多流连接**（路径数运行时确定）：声明 `activate_by_count`，强制将所有输入汇聚到同一个 Bucket，按字段计数达到阈值后触发。
- **静态计划与动态图**：解析阶段预计算边表、控制边、菱形根、关键槽等；运行时通过 NodeController 执行入站路由/出站分发，Bucket 状态机管理同步。
- **其他机制**：不可达传播与取消、跨图调用序列化（GCScheduler）、单线程事件循环避免竞态、基于 Redis 的服务发现与负载均衡（RoundRobin / SessionLoadBalance）。

#### 2.2 数据流（Data Flow）层
- **统一数据平面**：内存管理器（MemoryManager）作为服务端，客户端嵌入各角色进程，通过 ZMQ IPC 通信。
- **全局参数池**（Global Params Pool）：管理 KV 缓存的三级分页存储（GPU/CPU/SSD）。
  - 物理分页层：支持灵活组合（按 Transformer 层、每头切片），零拷贝获取张量视图。
  - 多级槽引擎：TinyLFU 计数 + 四层频率感知 LRU 驱逐；跨级迁移使用自定义 CUDA 内核转换格式；并发未命中去重。
  - 分布式元数据协调：Redis 作为真实源，Lua 脚本原子写、流水线读；布局标识通过 SHA-256 哈希；会话前缀匹配；副本发布与提供者选择；原子驱逐三层保护。
  - 数据流闭环：初始化→推理写→发布->预加载→跨节点 RDMA 拉取→写时复制与碎片页处理→内存压力下驱逐。
- **临时缓冲区池**（Tmp Buffer Pool）：预分配 slab + 引用计数 + LRU 双轨复用；跨主机采用拉取模式（pull-on-demand）而非推送，避免冗余传输。
- **权重共享**：monkey-patch PyTorch 工厂函数使参数分配到 meta device，之后通过 IPC 共享张量，多角色只占用一份 GPU 权重内存。
- **KV 缓存共享**：同 GPU 上多角色共享统一缓存池，屏障确保所有角色加载完权重后再确定池大小。

#### 2.3 计算流（Compute Flow）层
- **前缀匹配**：分解为全匹配（输入可重编码）和增量匹配（历史不可重编码）两种模式，共享链式分页哈希机制。LLM 前缀复用拆解为两层：层1（可重算的 emb）作为安全网，层2（KV 缓存）尽力而为。前缀匹配阶段按需产出多个帧以实现流水线预填充。
- **接口抽象与 KV 接管**：将 SGLang 运行时掏空，通过统一插件单例将控制权转给数据层。I/O 参数声明式，KV 缓存所有权在全局参数池，SGLang 注意力核只负责写入。
- **扩散模型复用 LLM 路径**：DiT 模块继承 LLM 的共享基类（KV 与权重管理），其去噪步骤沿标准分页注意力路径执行，使异构角色对齐并行切分语义（TP/EP），实现物理权重和 KV 缓存直接共享。
