## 摘要
无状态推理服务器（vLLM、SGLang、TensorRT-LLM）在请求之间空闲，而加速器等待；有状态会话则重新利用这段空闲时间。推测预定位使用目标模型自身的前向传播，无需草稿模型，将会话解码到下一个决策点，从而将跨请求的预填充和条目解码移出关键路径：下一个请求从其增量的预支付条目恢复，或者，当置信门触发时，从缓存的分布中以一次近乎恒定的词汇扫描直接回答，无需解码，代价仅为能量消耗和罕见的、有界的误接受。收益取决于模型能力：有能力的模型以近乎全覆盖和约87%的精度触发该门（较小的模型则永远无法清除），使得首个令牌返回时间约为1.0毫秒，而前缀缓存仍需支付39毫秒的解码时间。

## Abstract
A stateless inference server (vLLM, SGLang, TensorRT-LLM) idles between requests while the accelerator waits; a stateful session reclaims that idle time. Speculative pre-positioning decodes the session forward to its next decision point with the target model's own forward pass and no draft model, moving the cross-request prefill and entry-decode off the critical path: the next request resumes from a pre-paid entry on its delta, or, when a confidence gate fires, is answered from a cached distribution in one near-constant vocabulary scan with no decode, at a cost only of energy and a rare, bounded false accept. The payoff is conditional on capability: a capable model fires the gate at near-full coverage and about 87% precision (a smaller one never clears it), returning the first token in about 1.0 ms versus the 39 ms decode a prefix cache still pays.

## 1. 核心问题与整体含义（研究动机和背景）
- **问题**：无状态推理服务器（vLLM、SGLang、TensorRT-LLM）在请求之间空闲，加速器等待，造成计算资源浪费。传统有状态会话虽能保持状态跨请求，但 idle 窗口（如数据更新到下次查询、工具调用到结果返回之间）未被有效利用。
- **动机**：利用 idle 窗口提前完成下一个请求的入口工作（prefill + decode），从而将关键路径上的延迟移出，并允许在某些情况下直接以近乎零延迟返回 token。
- **整体含义**：提出“推测预定位”（Speculative Pre-Positioning），通过目标模型自身前向传播，在 idle 窗口解码会话到下一个决策点，并缓存输出分布。当请求到达时，要么只需增量恢复（fallthrough），要么在置信门触发时直接从缓存分布返回一个 token，无需任何额外解码。这是对现有有状态推理框架（Norgren 2026a, 2026b 的 streaming 和 tool-calling 系统）的优化。

## 2. 方法论
### 核心思想
- **决策点定义**：会话状态累积到 `n_s` token 后，下一个请求将附加一个长度 `L_entry` 的结构化入口（如流式会话的查询前缀+助手头，agentic 会话的 post-tool-call 包络）。决策点位置 `p = n_s + L_entry`。在该位置上进行一次前向传播即可得到“预备分布”（ready distribution）。
- **利用 idle 窗口**：状态达到版本 t 后、下一个请求到达前，加速器处于空闲。预定位的成本 `T_pp = T_prefill(L_entry)`，若满足 `T_pp ≤ I_idle`，则此工作对墙钟时间无影响，仅消耗能量。
- **快速路径**：若置信门（gate）触发，只需一次词汇扫描（vocabulary scan）即可返回 argmax token，关键路径延迟 `L_fast = T_scan(V)`（约 0.66 ms）。
- **回退路径**：若门未触发，请求从预支付的入口恢复，只需追加增量 ∆（如查询的具体文本），延迟 `L_fall = T_restore + T_prefill(∆) + T_decode`，优于冷路径。
- **置信门**：基于 top-1 与 top-2 logit 差距 `γ = ℓ(1) - ℓ(2)`，若 `γ > τ` 且预测 token 属于领域快速答案集 A，则触发。通过阈值 τ 调节覆盖率和假接受率的权衡（选择性预测框架）。
- **失效准则**：任何会话状态变异（滑动窗口移位、回滚、新更新）都会立即清除缓存分布的有效标志，确保不会服务过时答案。

### 关键公式
- 冷路径基线：`L_cold = T_prefill(L_entry) + T_decode`
- 快速路径：`L_fast = T_scan(V)`
- 回退路径：`L_fall = T_restore + T_prefill(∆) + T_decode`（∆ ≤ L_entry）
- 加速比：`S = (T_prefill(L_entry) + T_decode) / T_scan(V)`
- 净收益条件：当 `g*L_fast + (1-g)*L_fall < L_cold` 时，预期延迟降低。
- 命中率：`h = ρ/(1+ρ)`，其中 ρ = λ_q/λ_u（查询率/更新率）；`w = 1 - h` 为浪费率。
- 置信门风险-覆盖：覆盖率 c(τ)，假接受率 r(τ)，门触发当 `γ > τ ∧ argmax ∈ A`。

### 算法流程（文字描述）
- **Algorithm 1：空闲预定位**：会话更新后，若 cell budget 充足，则在末尾追加入口 token，执行一次前向传播得到预备分布 ℓ_ready，记录决策点 p、分布、有效性标志。
- **Algorithm 2：置信门快速路径服务**：请求到达时，检查有效性和匹配条件；若不满足，回退到调度器（正常路径，cost L_fall）；若满足，从缓存分布中取 top-2 logit 计算 γ，若 γ > τ 且预测 token 在 A 中，直接返回该 token（cost L_fast），否则回退。

## 6. 主要结论与发现
- **性能**：快速路径首次 token 延迟约 1.01 ms，比冷路径（53.1 ms）快 50 倍以上；相比前缀缓存（需 decode 约 39 ms）快约 40 倍。该加速在入口 token 较长时更明显（图 4）。
- **条件性**：收益取决于模型能力——70B 类模型在任务上自信，门以 0.945 覆盖率触发（87% 精度）；8B 模型门从不触发（零覆盖）。因此机制适用于“模型自信”的任务。
- **能量成本**：每次预定位约 63.8 J，在查询-更新平衡比（ρ=1）时，浪费能量约 191 W。
- **风险**：假接受率约 0.13（τ=2.0），且大部分（12/16）源于模型自身不确定性而非预承诺偏差。
- **通用性**：单一机制同时覆盖流式查询和 agentic 工具调用场景，且在失效准则下确保正确性。

## 7. 优点
- **创新性**：首次提出跨请求的 idle-window 预计算输出分布（而非仅 KV 缓存），并引入基于置信门的直接服务（无验证）。
- **分析清晰**：形式化决策点、延迟模型、命中率、净收益条件，将延迟分解成可解释组件图（图 5）。
- **实验可信**：测量分层进行，闭式分析与实测行为一致；提供 Wilson 区间、多个统计量。
- **实际可部署**：预定位是 best-effort、最低优先级，不会影响在线请求；失效准则保证正确性。
- **低开销**：仅需额外存储一个词汇表大小的向量（或最简形式 top token + logit gap），与 KV 缓存相比开销极小。

## 8. 不足与局限
- **任务单一**：仅验证了二分类市场趋势任务，未在更复杂数值或文本生成任务上测试（如多 token 答案、开放域问答）。
- **门完全依赖校准**：无验证步骤，若校准偏移（例如分布漂移），错误率直接上升；校准本身基于 128 个样本，可能不足以代表长期分布。
- **快速路径只适用于单 token 答案**：在 fast-answer set A 之外的查询只能回退，回退路径接近冷路径（因 entry 较短时 restore 成本抵消了预填充节省）。
- **模型依赖严重**：小模型或低置信任务上机制是 no-op（零覆盖），说明适用场景受限。
- **能量浪费**：更新密集型会话（ρ 小）浪费大多数预定位，能量效率低。
- **与先前技术不严格对比**：前缀缓存的延迟是推断而非实测，且未考虑实际系统中的并发与 batching 效应。
- **公平性**：70B 使用 4-bit，8B 使用 BF16，并非相同精度比较；未来需 BF16 70B 双卡验证。
- **仅限有状态会话**：不适用于无状态推理范式，因此不是通用加速技术。
- **未在高负载下测试**：未报告多个并发会话时的内存争用、调度优先级抢占对命中率的影响。
