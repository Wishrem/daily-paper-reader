## 摘要
大型语言模型越来越多地处理长上下文场景，此时KV缓存成为主要的内存瓶颈：其大小随序列长度线性增长，且必须在解码过程中全程保留，若不进行压缩，全GPU缓存的成本将高得令人望而却步。现有的KV缓存压缩方法难以在效率与忠实上下文保留之间取得平衡。词元驱逐会丢弃信息，而语义分组则在预填充阶段固定压缩决策；当生成过程中某个压缩片段变得相关时，这两种方法都无法从中恢复词元级别的细节。为此，我们提出SeKV，一种分辨率自适应的语义KV缓存，它将上下文组织为熵引导的语义片段，并跨GPU-CPU内存层级存储，不丢弃任何信息。每个片段在GPU上保留轻量级摘要向量用于粗粒度路由，在CPU上保留低秩SVD基用于按需词元级重建。训练得到的缩放机制在解码过程中选择性地扩展查询相关片段，从而无需在GPU上实例化完整KV缓存即可实现精确检索。SeKV支持自适应词元级重建，同时保持基础LLM完全冻结，仅增加不到0.05%的可训练参数。在四个基准测试中，SeKV相比最强的语义压缩基线平均提升5.9%，同时在128K上下文下相比全KV缓存减少53.3%的GPU内存。代码已开源：https://github.com/AmirAbaskohi/SeKV。

## Abstract
Large language models increasingly operate over long contexts, where the KV cache becomes a dominant memory bottleneck: its size grows linearly with sequence length and must be retained throughout decoding, making full GPU caching prohibitively expensive without compression. Existing KV cache compression methods struggle to balance efficiency with faithful context preservation. Token eviction discards information, while semantic grouping fixes compression decisions at prefill time; neither can recover token-level detail from a compressed span once it becomes relevant during generation. As a solution, we propose SeKV, a resolution-adaptive semantic KV cache that organizes context into entropy-guided semantic spans and stores them across a GPU-CPU memory hierarchy without discarding information. Each span keeps a lightweight summary vector on GPU for coarse routing and a low-rank SVD basis on CPU for on-demand token-level reconstruction. A trained zoom-in mechanism selectively expands query-relevant spans during decoding, enabling precise retrieval without materializing the full KV cache on GPU. SeKV enables adaptive token-level reconstruction while keeping the base LLM fully frozen and adding fewer than 0.05% trainable parameters. Across four benchmarks, SeKV improves over the strongest semantic compression baseline by 5.9% on average while reducing GPU memory by 53.3% versus full KV caching at 128K context. Code is available on https://github.com/AmirAbaskohi/SeKV.

## 2. 论文提出的方法论

### 核心思想
SeKV将上下文组织为**层级语义记忆**：将输入切分为语义连贯的片段（span），每个片段在GPU上保留紧凑的摘要向量用于粗粒度路由，在CPU上保留低秩SVD基用于按需token级重建。训练一个**缩放机制（zoom-in mechanism）**，在解码时选择性地展开与查询最相关的片段，从而实现自适应分辨率。

### 关键技术细节
1. **熵引导的片段分割（Entropy-Guided Span Segmentation）**
   - 利用预填充阶段token的**困惑度（surprisal）** $H_t = -\log p(x_t|x_{<t})$ 作为边界信号。
   - 高困惑度token通常标记语义转换（主题切换、实体引入等），将其作为**锚点token**（anchor tokens），保留在GPU上且始终以全分辨率参与注意力。
   - 相邻锚点之间的token构成一个语义片段$S$。片段长度自适应：信息密集区域片段短、锚点多；冗余区域片段长。
   - 阈值$\alpha$控制边界敏感度。

2. **双表示片段与内存层级**
   - **GPU端摘要向量**：对每个片段$S$，计算基于困惑度权重的投影平均键向量$\overline{K}_S^{(l,h)}$（维度$d' \ll d_h$），用于路由。
   - **CPU端低秩SVD基**：对每个片段的键矩阵$K_S^{(l,h)}$进行截断SVD（最大秩$R=32$），保留奇异向量和奇异值。通过一个轻量预测器$g_\phi$为每个组分分配软门控$m_i$，实现**自适应秩分配**：信息丰富的片段保留更多成分，冗余片段保留较少。
   - 值矩阵也采用类似处理。
   - 锚点token始终以全分辨率驻留GPU；短于$L_{\min}$的片段保留全分辨率。

3. **训练的缩放机制**
   - **路由阶段**：将当前查询$q^{(l,h)}$投影到路由空间，计算与每个片段摘要的Sigmoid相关性得分$\tilde{\alpha}_S^{(l,h)}$（含片段长度先验$\log|S|$），并与学习到的每头每层阈值$\tau^{(l,h)}$比较，决定是否展开该片段。
   - **展开阶段**：对选中的片段，异步从CPU提取SVD基，在GPU上重建token级KV对（公式8），并以全分辨率参与注意力。未展开的片段仅贡献一个粗粒度均值条目。
   - **损失函数**：包含四个项——蒸馏损失（对齐全KV教师）、缩放监督（二分类BCE）、重建损失（gated SVD误差）、预算损失（控制展开数和秩大小）。使用straight-through estimator处理二值决策。
   - **可训练参数**：仅包括路由投影矩阵$W^{(l,h)}$、阈值$\tau^{(l,h)}$和秩预测器$g_\phi$，总计约4.3M参数（占基础模型0.05%），基础LLM保持冻结。

### 公式与流程（文字说明）
- 预填充阶段计算每个token的困惑度，识别边界，划分片段，计算锚点、摘要向量，对每个片段的KV矩阵执行带门控的SVD分解并存储到CPU。
- 解码每一步：首先用投影查询与所有片段摘要计算相关性得分，通过阈值筛选出需要展开的片段；同时异步从CPU获取SVD基。将所有片段（锚点、未展开片段的均值、展开片段的重建token）合并到一个注意力计算中，输出结果。

## 7. 优点

- **动态分辨率**：不永久丢弃任何token，可根据查询需求按需恢复token级细节，克服了静态压缩的根本局限。
- **冻结LLM、极低额外参数**：仅增加0.05%可训练参数，便于部署。
- **熵引导分割无需外部工具**：利用预填充阶段已有的困惑度作为边界信号，无需额外模型或标注。
- **自适应秩分配**：通过门控SVD自动为每个片段分配最优压缩级别，信息密集区保留更多成分。
- **高效的内存层级**：GPU仅保留轻量摘要和锚点，CPU存储压缩SVD基，异步传输与计算重叠，延迟可控。
- **实验全面、消融充分、超参鲁棒**：覆盖多种模型、基准、预算和组件变体，结论可靠。

## 8. 不足与局限

- **依赖熵边界质量**：在高度碎片化的文档（表格、代码、非常规格式）中，困惑度可能无法准确标记语义边界，导致片段划分不佳，影响压缩效果。
- **引入多个超参数**：熵阈值$\alpha$、锚点数量、最大SVD秩$R$、本地窗口大小$w$等，在不同模型或上下文长度转移时需要调优验证。
- **CPU-GPU带宽瓶颈**：若生成过程中频繁展开大量远程片段（对抗性查询场景），传输开销可能增加延迟，削弱优势。
- **未联合训练LLM内部行为**：路由和缩放是仅为LLM外部添加的组件，LLM的注意力本身未适应压缩后的结构，可能限制了整体适应能力。
- **训练数据仅限语言建模**：虽覆盖多种领域，但未使用指令或检索数据，可能弱化跨分布泛化到特定下游任务的能力（但作者声称训练与评估数据无重叠，降低过拟合风险）。
- **未与最先进的量化和稀疏化方法联合比较**：例如量化（如KV量化）可能作为正交技术进一步降低内存，论文未评估结合效果。
