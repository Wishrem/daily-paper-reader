## 摘要
Transformer模块在大型语言模型（LLM）中普遍存在，但由于其苛刻的计算和内存需求，部署面临挑战。虽然先前的工作通常分别优化注意力机制或前馈网络（FFN），但很少有硬件架构通过协同设计的硬件加速来联合处理这两个组件。我们提出了ELiTeFormer（高效线性三元Transformer），这是首个将混合线性注意力与超低精度（三元）线性投影统一起来的Transformer模型架构，专门为现场可编程门阵列（FPGA）部署而协同设计。与LLaMA 3相比，ELiTeFormer实现了10倍的模型权重压缩和12.8倍的键值（KV）缓存压缩，同时保持有竞争力的精度（MMLU基准测试上为31.9%，在BitNet b1.58的3.0%以内）。我们关键的架构贡献是一种新颖的处理单元（PE）微架构，它通过位掩码操作消除了三元线性投影中的所有乘法，通过完全避免专用数字信号处理（DSP）块，显著减少了资源利用。我们使用高级综合（HLS）流程，针对Xilinx VCK5000 Versal板模拟、综合并部署了ELiTeFormer。块级模拟显示，与标准实现相比，FFN操作加速了9.6倍，注意力加速了4.4倍。在长上下文长度下，端到端部署相比在NVIDIA A100图形处理单元（GPU）上运行的LLaMA 3，实现了高达3.9倍的延迟降低和3.2倍的能效提升。这代表了首个将线性注意力与三元量化相结合的FPGA实现，展示了算法-架构协同设计用于下一代LLM加速的可行性。

## Abstract
Transformer blocks are prevalent in large language model (LLM) but present deployment challenges due to their challenging computational and memory demands. While prior work has typically optimized attention mechanisms or feed-forward networks (FFNs) separately, few hardware (HW) architecture have jointly addressed both components with co-designed hardware acceleration. We present ELiTeFormer (Efficient Linear Ternary Transformer), the first Transformer model architecture that unifies hybrid linear attention with ultra-low-precision (ternary) linear projections, specifically co-designed for field-programmable gate array (FPGA) deployment. ELiTeFormer achieves 10x model weight compression and 12.8x key-value (KV) cache compression compared to LLaMA 3, while maintaining competitive accuracy (31.9% on the MMLU benchmark, within 3.0% of BitNet b1.58). Our key architectural contribution is a novel processing element (PE) micro-architecture that eliminates all multiplications in ternary linear projections through bitmasking operations, significantly reducing resource utilization by completely avoiding dedicated digital signal processing (DSP) blocks. We simulate, synthesize, and deploy ELiTeFormer targeting a Xilinx VCK5000 Versal board using high-level synthesis (HLS) flows. Block-level simulations show 9.6x speedup for FFN operations and 4.4x speedup for attention compared to standard implementations. End-to-end deployment achieves up to 3.9x lower latency and 3.2x better energy efficiency than LLaMA 3 on an NVIDIA A100 graphics processing unit (GPU) at long context lengths. This represents the first FPGA realization combining linear attention with ternary quantization, demonstrating the viability of algorithm-architecture co-design for next-generation LLM acceleration.

## 1. 论文的核心问题与整体含义（研究动机和背景）
Transformer 模块（特别是注意力机制和前馈网络 FFN）是大型语言模型（LLM）的核心，但在 FPGA 等受限硬件上部署时面临严峻的计算和内存挑战。现有工作通常**分别优化**注意力（如稀疏/线性注意力）或 FFN（如低精度量化），但**缺乏联合的算法-硬件协同设计**。同时，FPGA 上的 LLM 加速器多沿用传统 Transformer 架构，难以利用高效注意力变体和超低精度运算的优势，且存在静态编译对动态生成过程（如无界生成）支持不足的问题。ELiTeFormer 旨在通过**算法-架构联合设计**，首次在 FPGA 上实现融合混合线性注意力和三元线性投影的高效 Transformer。

## 2. 论文提出的方法论
### 核心思想
- **算法层面**：将 **混合线性注意力**（Hedgehog + 滑动窗口）与 **三元量化线性投影**（BitNet b1.58）统一到单一 Transformer 架构中，实现模型权重 10× 压缩、KV 缓存 12.8× 压缩。
- **硬件层面**：设计了一种**无乘法器的处理单元（PE）微架构（ELTF PE）**，通过位掩码操作代替三元权重（值域 {-1,0,1}）与激活值的乘法，完全避免使用 FPGA DSP 块。

### 关键技术细节
- **模型结构**：基于 LLaMA 3 8B 架构，将标准自注意力替换为**混合注意力**——并行执行 Hedgehog 线性注意力（带可学习特征映射）和滑动窗口稀疏注意力（窗口大小 256 token）。所有线性投影（Q、K、V、输出、FFN）均采用 BitNet b1.58 风格的三元量化，仅在注意力特征映射部分保留全精度。
- **训练流程**：采用 **LoLCATs** 注意力蒸馏方法，分两阶段：
  1. **逐层替换+蒸馏**：冻结除新注意力块外的所有权重，以输出重构误差最小化为目标逐层训练（MSE 损失）。
  2. **轻量微调**：尝试添加 LoRA 适配器，但发现 LoRA 对三元模型不利，反而降低性能（Table 2），故最终省略该步骤。
- **硬件 PE 微架构**（图 3）：PE 内每个时钟周期读入 4 个输入值和 1 个 8-bit 权重数据帧；将权重数据帧位移为 4 个 2-bit 索引，每个索引选择对应掩码（正、零、负），通过位与/或操作实现三元乘法模拟；结果经累加树合并到输出值中。整个设计无需乘法器，仅需 LUT 和寄存器。

### 公式/算法流程（文字说明）
- 混合注意力计算：`output = LinearAttention(Q,K,V) + SlidingWindowAttention(Q,K,V)`，其中线性注意力采用狸猫式（Hedgehog）核函数近似，滑动窗口限制局部范围。
- 线性投影（如 `Y = X · W^T`）在硬件中分解为：对每个输入元素，根据三元权重 {-1,0,1} 执行 `+x`, `0`, `-x` 操作，并用累加树求和。

## 7. 优点
1. **算法-架构协同创新**：首次在 Transformer 中联合应用贝叶斯线性注意力和三元量化，并为其定制无乘法器 PE。
2. **极低硬件资源**：ELTF PE 完全避免 DSP，仅用 LUT 和 BRAM，适合资源受限的 FPGA。
3. **静态推理内存**：混合线性注意力在生成阶段内存占用与上下文长度无关，简化 FPGA 编译和部署。
4. **极端压缩**：权重和 KV 缓存压缩比远超现有 FPGA 加速器（如 FlightLLM）。
5. **全面的实验评估**：覆盖质量、延迟、吞吐、能效、资源利用率，且提供模拟与真机部署双重验证。

## 8. 不足与局限
1. **模型质量下降显著**：MMLU 31.9% 远低于 LLaMA 3（66.6%），在 HellaSwag、ARC-c 等任务上退化明显，仅与 BitNet b1.58 水平接近。
2. **训练数据不足**：仅使用 50k Alpaca 样本微调，且预训练 checkpoint 为 BitNet b1.58（本身已在 100B tokens 上训练），但未与等量数据训练的线性注意力模型公平比较。
3. **滑动窗口限制**：窗口大小仅 256 token，可能无法捕获长距离依赖，影响中等长度上下文任务。
4. **硬件部署规模小**：仅实现 1B 参数模型（6 层，hidden dim 4096），8B 模型仅做模拟，实际 FPGA 资源能否支撑更大规模未验证。
5. **对比基线偏差**：GPU 基线可能未充分利用 Tensor Core 的极低精度运算（BitNet b1.58 框架 bitnet.cpp 已优化，但未使用 W4A4 等问题）；能效比较未考虑 FPGA 板卡功耗以外的系统开销。
6. **LoRA 无效**：论文发现 LoRA 对三元模型有害，但未探索其他适配方法（如 AdaLoRA），可能导致模型的跨层误差鲁棒性不足。
