## 摘要
推测解码通过将草稿生成与目标验证解耦，加速了大语言模型（LLM）的推理过程。尽管最近的并行草稿生成器能够在单次前向传播中高效地生成长令牌序列，但由于缺乏令牌间依赖关系，它们会遭受快速接受衰减的问题。此外，不加区分地验证这些扩展块会浪费关键批处理容量在具有高拒绝风险的令牌上，严重降低高并发服务系统的吞吐量。我们提出了DSpark，一个统一的推测解码框架，将高吞吐量的并行生成与自适应、负载感知的验证相结合。为了保持草稿质量，DSpark采用半自回归架构，将并行骨干网络与轻量级序列模块耦合，引入模块内依赖建模并缓解后缀衰减。为了优化系统效率，DSpark采用基于置信度调度的验证，根据估计的前缀生存概率和引擎特定的吞吐量特征，动态调整每个请求的验证长度。在跨不同领域的离线基准测试中，DSpark相比最先进的自回归和并行草稿生成器显著提升了接受长度。当在实时用户流量下部署于DeepSeek-V4服务系统时，DSpark成功缓解了验证浪费。与已建立的生产基线（MTP-1）相比，DSpark在匹配的吞吐量水平下将每用户生成速度提升了60%至85%。更重要的是，通过防止在严格交互性约束下严重的吞吐量下降，它实现了以前无法达到的性能层级，推动了服务系统的帕累托前沿。

## Abstract
Speculative decoding accelerates Large Language Model (LLM) inference by decoupling draft generation from target verification. While recent parallel drafters efficiently propose long token sequences in a single forward pass, they suffer from rapid acceptance decay due to a lack of inter-token dependencies. Furthermore, indiscriminately verifying these extended blocks wastes critical batch capacity on tokens with high rejection risks, severely degrading throughput in high-concurrency serving systems. We introduce DSpark, a speculative decoding framework that unifies high-throughput parallel generation with adaptive, load-aware verification. To maintain draft quality, DSpark utilizes a semi-autoregressive architecture, coupling a parallel backbone with a lightweight sequential module, to introduce intra-block dependency modeling and mitigate suffix decay. To optimize system efficiency, DSpark employs confidence-scheduled verification, dynamically tailoring the verification length for each request based on estimated prefix survival probabilities and engine-specific throughput profiles. On offline benchmarks across diverse domains, DSpark substantially improves the accepted length over state-of-the-art autoregressive and parallel drafters. When deployed within the DeepSeek-V4 serving system under live user traffic, DSpark successfully mitigates verification waste. Compared to the established production baseline (MTP-1), DSpark accelerates per-user generation speeds by 60 to 85 percent at matched throughput levels. More importantly, by preventing severe throughput degradation under strict interactivity constraints, it enables performance tiers that were previously unattainable, shifting the Pareto frontier of our serving system.

## 1. 论文的核心问题与整体含义（研究动机和背景）

- **核心问题**：大语言模型（LLM）自回归生成每个令牌需要一次完整前向传播，导致推理延迟与输出长度成正比，GPU利用率低，用户等待时间长。推测解码虽能加速，但现有方法面临两个瓶颈：
  1. **生成质量瓶颈**：并行草稿生成器（如DFlash）因缺乏模块内令牌间依赖关系，在多模态上下文下易产生不连贯组合（如“of problem”），导致接受率随位置快速衰减。
  2. **系统效率瓶颈**：不加区分地验证长草稿块会浪费批处理容量在低置信度后缀令牌上，尤其在高并发场景下会耗散本可用于其他请求的计算资源。

- **整体含义**：需要一种既能保持并行生成高吞吐（草稿延迟几乎不随块大小增长），又能引入模块内依赖以提高接受率，同时根据系统负载动态调整验证长度的方法，以同时优化生成质量和系统吞吐。

## 2. 论文提出的方法论：核心思想、关键技术细节

### 核心思想
DSpark 采用**半自回归生成架构**（并行骨干 + 轻量级序列模块）提高草稿质量，并引入**置信度调度验证**（confidence-scheduled verification）根据实时负载动态修剪验证长度。

### 关键技术细节
#### （1）半自回归生成
- **并行阶段**：使用 DFlash 骨干（并行，单次前向传播生成所有草稿隐状态和基础 logits）。改进：以锚令牌（上一轮目标模型生成的最后一个令牌）作为第一个预测位置，减少计算量。
- **序列阶段**：在基础 logits 上叠加前缀依赖的转移偏置，通过自回归分解实现因果块分布：
  \[
  P(X|x_0) = \prod_{k=1}^{\gamma} p_k(x_k|x_0, x_{<k}),\quad p_k(v|\cdot) = \frac{\exp(U_k(v) + B_k(\cdot, v))}{\sum_u \exp(U_k(u) + B_k(\cdot, u))}
  \]
  两个实例化：
  - **Markov头**：仅依赖前一个令牌，用低秩分解 \(B = W_1W_2\)（\(r=256\)）近似转移矩阵，高效。
  - **RNN头**：维护循环状态 \(s_k\)，累积前缀历史，用单个门控更新生成转移偏置。

#### （2）置信度调度验证
- **置信度头**：轻量线性投影 + sigmoid，输出每个位置 \(c_k\) 表示给定前面所有令牌被接受条件下该位置被接受的条件概率。监督信号为理论接受率 \(c_k^* = 1 - \frac12\|p_k^d - p_k^t\|_1\)。
- **事后校准**：引入**序列温度缩放（STS）**，从左到右依次校准每个位置的累积接受概率，最小化期望校准误差（ECE）。
- **硬件感知前缀调度器**：将验证长度选择建模为全局吞吐最大化问题。每个请求 \(r\) 有前缀生存概率 \(a_{r,j} = \prod_{i\le j} c_{r,i}\)，总期望接受令牌数 \(\tau = \sum_r (1 + \sum_{j=1}^{\ell_r} a_{r,j})\)，目标模型步吞吐 SPS(\(B\)) 通过预分析曲线得到。算法贪心选择全局最优验证长度：对所有 \(a_{r,j}\) 排序后依次添加，直到吞吐不再增加。为保证无损（非预期性），采用早停机制：一旦吞吐下降就停止，避免依赖未来令牌信息。

#### 训练目标
三部分加权：
- 交叉熵损失 \(\mathcal{L}_{\text{ce}}\)（预测正确令牌）
- TV距离损失 \(\mathcal{L}_{\text{tv}}\)（最小化草稿与目标分布的总变差）
- 置信度损失 \(\mathcal{L}_{\text{conf}}\)（二元交叉熵）
所有损失按位置指数衰减加权（\(w_k = \exp(-(k-1)/\gamma)\)），强调早期位置。
