## 摘要
长上下文LLM推理日益受到KV缓存的内存和带宽成本的限制，但激进的压缩会去除检索和多步推理所需的逐层证据。我们提出FreqDepthKV，一种推理时缓存压缩方法，将相邻层的KV状态分解为共享的低频深度分量和稀疏的高频残差。轻量级在线探针根据注意力头对重建敏感注意力logits的贡献，将其分配至共享深度、残差深度或精确缓存模式，使得压缩策略无需重新训练即可适应提示结构。在长上下文问答、针检索、摘要和代码生成基准测试中，FreqDepthKV在显著更小的缓存预算下保持了任务准确性。在32k token预填充窗口下，FreqDepthKV达到了58.3 Exact Match、63.0 F1、32.5 ROUGE-L和48.1 pass@1，与全KV缓存紧密匹配，同时优于先前的压缩缓存方法。它还将解码吞吐量提升至70.4 tokens/s，将首token延迟（TTFT）降低至2.06秒，并将峰值KV内存降低至6.2 GB，实现了3.9倍的有效压缩比。

## Abstract
Long-context LLM inference is increasingly limited by the memory and bandwidth cost of KV caches, yet aggressive compression can remove the layer-specific evidence needed for retrieval and multi-step reasoning. We introduce FreqDepthKV, an inference-time cache compression method that factorizes adjacent-layer KV states into shared low-frequency depth components and sparse high-frequency residuals. A lightweight online probe assigns attention heads to shared-depth, residual-depth, or exact cache modes according to their contribution to reconstruction-sensitive attention logits, allowing the compression policy to adapt to prompt structure without retraining. Across long-context question answering, needle retrieval, summarization, and code generation benchmarks, FreqDepthKV preserves task accuracy under substantially smaller cache budgets. With a 32k-token prefill window, FreqDepthKV reaches 58.3 Exact Match, 63.0 F1, 32.5 ROUGE-L, and 48.1 pass@1, closely matching full KV while outperforming prior compressed-cache methods. It also improves decoding throughput to 70.4 tokens/s, reduces TTFT to 2.06 seconds, and lowers peak KV memory to 6.2 GB, achieving a 3.9x effective compression ratio.

## 1. 核心问题与整体含义（研究动机和背景）

- **背景**：长上下文LLM推理中，KV缓存成为主要内存和带宽瓶颈。自回归解码时，每生成一个token都需要访问之前所有token的KV缓存，导致内存消耗随上下文长度线性增长。
- **动机**：现有压缩方法（如token驱逐、量化、均匀深度共享）能减少内存，但往往将冗余视为均匀现象，忽略了相邻层间缓存结构的频率特性。尤其是过度压缩会删除检索和推理所需的层特定证据（如“针”句子、代码依赖），导致下游注意力logits偏移，即使重建误差小也会改变答案。
- **整体含义**：FreqDepthKV旨在通过频率引导的深度共享，在保留对检索敏感的高频残差的同时压缩冗余深度信息，实现无需重训练即可适应提示结构的鲁棒缓存压缩。

## 2. 方法论：核心思想、关键技术细节

### 核心思想
- 将相邻层的KV缓存分解为**低频共享深度分量**（广泛共享）和**稀疏高频残差**（保留检索敏感证据）。
- 利用轻量在线探针，为每个注意力头分配三种缓存模式：**shared-depth**（仅低频）、**residual-depth**（低频+稀疏残差）、**exact**（不压缩），根据重建注意力logits的误差来决定。

### 关键技术细节
1. **深度频率分解**：对于每B个相邻层（默认B=4，首尾层用B=2），沿深度轴堆叠KV，施加固定DCT变换，得到系数。第一个系数作为低频共享分量，其余为高频残差。
2. **在线头路由**：预填时，对每个头，用采样查询位置P计算三种模式下的重建注意力logits与其原始值之间的L2距离，加上正则化内存代价，选择使损失最小的模式。
3. **稀疏残差选择**：对于residual-depth模式，根据移除高频系数后引起的最大logit扰动评分每个token，保留top-r比例token的高频残差。
4. **推理时使用**：解码时，共享系数广播到各层，残差系数只添加到索引到的token，避免实例化全缓存。
5. **预算控制**：通过调整内存惩罚λ使估计峰值KV内存满足预算。
