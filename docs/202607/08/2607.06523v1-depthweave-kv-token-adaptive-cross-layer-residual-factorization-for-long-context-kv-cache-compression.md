## 摘要
长上下文语言模型推理日益受到存储键值缓存所需内存带宽和容量的限制，然而现有压缩方法通常在各层或各令牌间应用统一预算，并在词法线索和语义状态需要不同保留时降低检索质量。我们提出DepthWeave-KV，一种令牌自适应缓存压缩方法，该方法使用共享的低秩通道基对相邻Transformer层的键和值状态进行分解，同时在注意力行为敏感的令牌处保留轻量级特定令牌残差。DepthWeave-KV结合跨深度残差分解与令牌条件深度路由，为承载指令和检索关键的令牌分配更高的重建秩，并利用注意力输出探针的免校准在线错误跟踪，在生成过程中自适应压缩，无需重新训练基础模型。融合的CUDA实现联合执行基查找、残差反量化和注意力投影，以减少解码时的内存流量。在LongBench、Needle-in-a-Haystack、L-Eval以及长形式问答和摘要基准测试中，DepthWeave-KV在显著降低内存使用的情况下实现了接近完整缓存的任务质量，较之前的压缩缓存提高了平均得分和检索准确率，同时在64K上下文下实现了8.3倍的KV内存缩减和每秒72.8个令牌的处理速度。

## Abstract
Long-context language model inference is increasingly limited by the memory bandwidth and capacity required to store key-value caches, yet existing compression methods often apply uniform budgets across layers or tokens and degrade retrieval when lexical cues and semantic states require different preservation. We introduce DepthWeave-KV, a token-adaptive cache compression method that factorizes key and value states across neighboring transformer layers using shared low-rank channel bases while retaining lightweight token-specific residuals where attention behavior is sensitive. DepthWeave-KV combines cross-depth residual factorization with a token-conditional depth router that allocates higher reconstruction rank to instruction-bearing and retrieval-critical tokens, and uses calibration-free online error tracking from attention-output probes to adapt compression during generation without retraining the base model. A fused CUDA implementation jointly performs basis lookup, residual dequantization, and attention projection to reduce decode-time memory traffic. Across LongBench, Needle-in-a-Haystack, L-Eval, and long-form QA and summarization benchmarks, DepthWeave-KV achieves near-full-cache task quality with substantially lower memory use, improving average score and retrieval accuracy over prior compressed caches while reaching 8.3x KV memory reduction and 72.8 tokens per second at 64K context.

- **问题**：长上下文语言模型推理中，KV缓存成为内存带宽和容量的主要瓶颈，限制了批大小、上下文长度和服务吞吐量。现有压缩方法（如令牌驱逐、合并、量化、低秩注意、层共享）通常对所有层或令牌应用**统一预算**，忽略了不同层（浅层编码词法线索、深层编码语义抽象）和不同令牌（指令分隔符、实体提及、检索关键跨度 vs. 流畅续写令牌）对缓存保真度的异质性需求，导致在需要精确检索的任务（如 Needle-in-a-Haystack、多跳QA）中性能严重退化。
- **动机**：深度方向存在大量冗余（MiniCache等已证明），但简单共享不够；需要同时考虑深度和令牌重要性，动态分配重建容量，且能不依赖校准数据或重训练基座模型进行在线自适应压缩。

## 2. 方法论
### 核心思想
- **跨深度残差分解**：将相邻Transformer层的KV状态分解为**共享的低秩通道基**（per attention head）和**令牌特定的量化残差**，并引入轻量级残差门控。
- **令牌条件深度路由器**：根据实时特征为关键令牌分配更高的残差秩，同时压缩低重要性令牌。
- **免校准在线误差跟踪**：通过计算压缩重建与高保真缓存的注意力输出差异来动态调整压缩比率。

### 关键技术细节
1. **跨深度残差分解**（式1）：
   - 将层划分为重叠深度窗口 \(W_m\)，每个窗口每头维护共享基 \(B_{m,h} \in \mathbb{R}^{r_b \times d}\)。
   - 对于令牌 \(t\) 在层 \(\ell\) 的KV状态重建：\(\hat{X}_{\ell,h,t} = A_{\ell,h,t} B_{m,h} + g_{\ell,h,t} R_{\ell,h,t}\)，其中 \(A\) 是系数向量（8-bit量化），\(R\) 是量化残差（4-bit分组量化），\(g\) 是残差门控∈[0,1]。
   - 基在预填充后通过随机低秩草图初始化，生成过程中用指数移动平均更新。

2. **令牌条件深度路由器**（式2）：
   - 路由特征 \(\phi_{m,h,t}\) 包括：累积注意力质量、最大注意力尖峰、分隔符/指令边界指示器、最近探针间隔的注意力输出重建误差。
   - 输出残差秩 \(\rho = \rho_{\min} + (\rho_{\max} - \rho_{\min}) \cdot \mathbf{1}[\sigma(u^\top \phi) > \tau_m]\)，实际使用三级（0/2/4/8）。
   - 路由器参数 \(u\) 仅需一次离线训练，部署时由在线误差控制器调整阈值 \(\tau_m\)。

3. **在线误差跟踪**（式3）：
   - 每 \(p\) 个解码步，选取小部分头和最近查询位置，分别用压缩重建缓存和临时高保真缓存计算注意力输出，得到归一化误差 \(e_{\ell,h}\)。
   - 若 \(e\) 超过容差，则降低当前窗口阈值，允许更多令牌使用更高残差秩；反之提高阈值。

4. **融合CUDA实现**：
   - 核函数融合基查找、残差反量化、系数乘法和注意力计算，重建的KV片段不写回全局内存，减少解码时内存流量。

- **无需重训练基座模型**：仅残差门和路由器参数需预训练，部署时在线调整无需校准数据。
