## 摘要
我们提出了Nemotron-Labs-3-Puzzle-75B-A9B，这是Nemotron-3-Super的压缩变体，针对交互式部署进行了优化。我们设计该模型以在高用户吞吐量约束下最大化服务器吞吐量。在单个8xB200节点上的交互式服务工作负载中，Puzzle-75B-A9B在匹配的用户吞吐量约束下比Nemotron-3-Super实现了约2倍的服务器吞吐量提升。在单个H100 GPU上的超长上下文部署中，压缩模型将1M令牌并发从1个请求增加到8个请求。Puzzle-75B-A9B通过多阶段流水线构建，该流水线结合了迭代拼图压缩框架与知识蒸馏、强化学习、量化和多令牌预测头。压缩过程联合优化了异构MoE剪枝、活跃参数预算和Mamba剪枝，以提高推理效率同时保持模型质量。我们在广泛的推理、编码、多语言、长上下文和智能体基准上评估了Puzzle-75B-A9B。尽管大幅压缩，该模型在广泛任务中相对于父模型仍保持了强大的下游准确性。这些结果表明，大型混合MoE模型可以在保持强大下游能力的同时显著优化部署效率。我们的模型已在Hugging Face上公开发布。

## Abstract
We present Nemotron-Labs-3-Puzzle-75B-A9B, a compressed variant of Nemotron-3-Super optimized for interactive deployment. We designed the model to maximize server throughput under high user throughput constraints. In interactive serving workloads on a single 8xB200 node, Puzzle-75B-A9B achieves approximately 2x higher server throughput than Nemotron-3-Super at matched user throughput constraints. In ultra-long-context deployment on a single H100 GPU, the compressed model increases 1M-token concurrency from 1 request to 8 requests. Puzzle-75B-A9B is constructed using a multi-stage pipeline that combines the Iterative Puzzle compression framework with knowledge distillation, reinforcement learning, quantization, and a Multi-Token Prediction head. The compression process jointly optimizes heterogeneous MoE pruning, active parameter budget, and Mamba pruning to improve inference efficiency while preserving model quality. We evaluate Puzzle-75B-A9B on a broad suite of reasoning, coding, multilingual, long-context, and agentic benchmarks. Despite substantial compression, the model retains strong downstream accuracy relative to the parent model across a wide range of tasks. These results demonstrate that large hybrid MoE models can be substantially optimized for deployment efficiency while maintaining strong downstream capability. Our model is publicly available on Hugging Face.

## 2. 论文提出的方法论

本工作提出了一套多阶段压缩与恢复流水线，以最大化效率提升。

### 核心思想：迭代式硬件感知架构搜索与性能恢复

### 关键技术细节：

-   **迭代 Puzzle 框架 (Iterative Puzzle)**：一种顺序压缩方法。它将传统的单次 Puzzle 压缩扩展为**多轮压缩-恢复阶段**。在每一轮中，Puzzle 在当前已压缩模型的上下文内搜索，并重新计算候选架构的评分，从而更好地捕捉层间交互与依赖关系，最终通过多次迭代达到目标架构。这避免了一次性压缩带来的次优解。
-   **异构 MoE 剪枝**：
    -   **中间通道剪枝 (Intermediate channel pruning)**：在每个路由专家的内部维度上进行剪枝，减少活跃参数和总参数量。不同层可以使用不同的剪枝率。
    -   **Top-k 剪枝 (Top-k reduction)**：允许不同 MoE 层使用不同数量的活跃专家（k），以在计算关键阶段（prefill/大batch解码）减少活跃参数。
-   **Mamba SSM 状态剪枝 (Mamba SSM Pruning)**：通过评估每个 SSM 通道对整个 Mamba 层输出的**贡献度**，将 SSM 状态大小从 128 剪枝到 96。这种方法在保留关键信息的同时，有效加速了 Mamba 层的解码速度（1.2x-1.3x）。

-   **性能恢复技术 (Recovery Phase)**：
    -   **知识蒸馏 (KD)**：在迭代 Puzzle 阶段和最终恢复阶段，压缩后的“学生”模型学习原始“教师”模型（Nemotron-3-Super）的 logits。使用混合数据集（30% 预训练数据，70% SFT 数据），序列长度从 32K 逐步扩展到 128K 和 512K，训练规模高达 1000 亿 token。
    -   **强化学习 (RL)**：针对在压缩中退化最显著的**软件工程**能力进行特定恢复。RL 分两个阶段（单步工具使用和端到端沙盒评估），采用**权重平均（Weight Averaging）**策略，通过对不同学习率训练的多个检查点进行权重平均，以降低方差并提高稳定性。
-   **量化与推理加速**：
    -   **模型量化**：为部署在 Blackwell 和 Hopper GPU，分别生成了 **NVFP4 (W4A4)** 和 **FP8 (W8A8)** 精度的检查点。针对不同算子（如 Mamba 线性层、KV cache、路由器等）采用了差异化的量化精度策略，以平衡精度和效率。
    -   **多令牌预测 (Multi-Token Prediction, MTP)**：模型原生支持预测多个未来 token，并通过**共享头 (shared-head)** 设计，使其在推理时可以被递归地应用以生成长草稿。论文还针对性地改进了 MTP 训练，使其更好地匹配自回归推理，显著提升了接受长度（约 25-30%）。
-   **最终架构**：从 `Nemotron-3-Super` (120.7B 总/12.8B 活跃) 压缩到 `Puzzle-75B-A9B` (75.3B 总/9.3B 活跃)。保留了 88 个原始块（40 Mamba + 40 MoE + 8 Attention），但 MoE 层的容量分布是**高度异构的，根据层的重要性进行分配**。
