## 摘要
多模态大语言模型必须适应不断变化的任务和领域，但在有限的部署占用下实现持续改进仍然困难，因为重复的参数更新或不断增长的存储池可能会随时间累积适应状态。我们研究固定占用的持续适应：部署的适应状态保持在固定的内存预算下，而骨干模型保持不变，任务特定的更新被外部化。我们提出InduceKV，一种基于检索的方法，它将每个选定的训练前缀存储为一个注意力就绪的记忆条目，由一个冻结的检索键和紧凑的逐层键-值（KV）负载组成，这些负载可以附加到模型的自注意力缓存中。在严格的内存预算下，InduceKV通过双层选择构建一个紧凑的诱导集：轻量级校准适用于检索，而选定的记忆在当前任务似然、基于锚点的保留以及冻结检索空间中的覆盖之间取得平衡。在任务增量指令调优、持续VQA、领域增量适应和终身多模态指令调优中，InduceKV在匹配的内存预算下始终优于PEFT、MoE、重放和提示检索基线。我们进一步报告了骨干匹配、阶段1 CoIN、计算匹配和可扩展性诊断，表明这些增益并非来自更强的骨干、单独的重放或无界的候选池。

## Abstract
Multimodal large language models must adapt to evolving tasks and domains, yet continual improvement under bounded deployment footprint remains difficult because repeated parameter updates or growing replay stores can accumulate adaptation state over time. We study fixed-footprint continual adaptation: the deployed adaptation state is kept under a fixed memory budget, while the backbone model is left unchanged and task-specific updates are externalized. We propose InduceKV, a retrieval-based method that stores each selected training prefix as an attention-ready memory entry, consisting of a frozen retrieval key and compact layerwise key--value (KV) payloads that can be appended to the model's self-attention cache. Under a strict memory budget, InduceKV constructs a compact inducing set through bilevel selection: a lightweight calibration is fit for retrieval, while the selected memory balances current-task likelihood, anchor-based retention, and coverage in the frozen retrieval space. Across task-incremental instruction tuning, continual VQA, domain-incremental adaptation, and lifelong multimodal instruction tuning, InduceKV consistently improves over PEFT, MoE, replay, and prompt-retrieval baselines under matched memory budgets. We further report backbone-matched, stage-1 CoIN, compute-matched, and scalability diagnostics, showing that the gains are not due to a stronger backbone, replay alone, or an unbounded candidate pool.

### 2. 论文提出的方法论：核心思想、关键技术细节

#### 核心思想
- **冻结MLLM基础模型**，不更新其参数。
- 将每个训练前缀**提取为记忆条目**：一个归一化的检索键（用于索引）+ 紧凑的逐层KV负载（可直接注入自注意力缓存）。
- 在**固定条目预算**下，通过**双层优化**从候选池中选出一个紧凑的诱导集，使选出的记忆能同时优化**当前任务似然**、**历史锚点保留**以及**检索空间的低冗余覆盖**。

#### 关键技术细节
1. **条目提取**：对每个前缀输入 \(x\)，用冻结模型计算：
   - **检索键** \(r(x) = \bar{h}(x)/\|\bar{h}(x)\|\)（最终层隐藏状态的均值归一化）。
   - **层KV负载**：每层对前缀令牌计算原始 \(K, V\)，再通过固定池化矩阵 \(P_m\) 压缩到固定长度 \(m\)，得到 \(\bar{K}_\ell(x), \bar{V}_\ell(x)\)。
2. **注入机制**：推理时，根据查询前缀的检索键 \(r(x)\) 与记忆条目的余弦相似度，用可学习温度 \(\tau\) 计算权重 \(\alpha_i\)。然后每层将记忆KV负载**加权拼接**到原自注意力缓存中，并通过**可学习逐层值门控** \(\lambda_\ell\) 缩放。
3. **双层选择**（给定固定预算 \(B\)）：
   - **内层**：固定当前选择权重 \(w\)，优化校准参数 \(\phi = \{\tau, \lambda_\ell\}\) 以最小化当前任务负对数似然 + L2 正则。
   - **外层**：交替更新选择权重 \(w\)（在预算单纯形上投影梯度下降），目标为：  
     \[
     J(w) = \mathcal{L}_{\text{cur}}(w, \phi^*) + \beta \mathcal{L}_{\text{anc}}(w, \phi^*) + \gamma \Omega_{\text{spec}}(w)
     \]
     - \(\mathcal{L}_{\text{cur}}\)：当前任务似然损失。
     - \(\mathcal{L}_{\text{anc}}\)：**历史锚点损失**（用少量锚点代理旧任务风险）。
     - \(\Omega_{\text{spec}}(w)\)：**log-det 覆盖正则**，在固定投影子空间上惩罚检索键的冗余（低秩集中）。
   - 收敛后取 Top-B 权重对应的条目作为最终记忆 \(M_t\)。

4. **理论支撑**（Section 4）：
   - 将记忆更新建模为**在线预算子集选择**问题，证明在松弛的权重更新下，静态遗憾和动态遗憾均可达到 \(O(\sqrt{T})\) 界，且锚点损失能够近似历史风险。

### 6. 论文的主要结论与发现

- **InduceKV 在固定内存预算下，在所有评估场景中一致优于现有方法**（PEFT、MoE、重放、提示检索），且无需更新骨干。
- **增益来源于诱导集的高效选择**：双层优化平衡了当前任务拟合、历史保留（锚点）和检索空间覆盖，构建出紧凑且非冗余的记忆。
- **记忆确实被有效利用**：注入的KV记忆在生成长序列时被注意力机制积极利用（尤其中间到层），且高增益任务对应更强的记忆注意力。
- **鲁棒性强**：对任务顺序、超参数变化、不同骨干（从1B到8B）均表现稳定。
- **理论支撑**：在线预算子集选择具有次线性遗憾，锚点准确代理历史风险。
- **实践上**：虽然推理预填充略有开销（额外一次检索键计算），但解码吞吐量接近重放无关方法，且远优于提示级检索。

### 7. 优点

1. **创新性**：将持续适应重新定义为**固定预算诱导集在线选择**，将稳定性–可塑性矛盾转化为内存分配问题，而非不断增长的参数更新。
2. **保持骨干冻结**：不扰乱预训练的跨模态交互，易于审计和分解。
3. **理论完备**：给出静态/动态遗憾界以及锚点近似历史风险的理论保证。
4. **充分的实验验证**：多场景、多模型、多项消融和机制分析，证明有效性来源。
5. **公平对比**：严格匹配占用（按FP16字节、参数量或存储字节）和推理计算。
6. **可扩展性**：短列表候选池可显著降低更新开销，几乎不损失质量。

### 8. 不足与局限

1. **推理预填充开销**：相比纯PEFT（即无外部检索），需要一次额外的前缀前向传递以计算检索键，预填充延迟增加。
2. **依赖模型内部**：需要访问冻结模型的隐藏状态和KV缓存接口，对非标准注意力或受限API的模型需要额外工程适配。
3. **记忆含训练信息**：KV条目由训练前缀导出，需与原始数据同等级别的隐私保护；虽然可独立删除，但未直接解决隐私问题。
4. **锚点假设**：虽然实验证明锚点损失与历史风险高度相关（Pearson 0.94），但理论假设锚点具有代表性（ϵ-近似），在实践中可能不总是满足（如任务大幅偏移时）。
5. **实验范围**：所有实验在公开学术基准和检查点上进行，对更复杂、更大规模的实际部署场景（如多模态多轮对话）有待验证。
6. **计算资源**：虽然不需要训练骨干，但双层选择和条目提取仍需要一定的GPU时间（全项目约1100 A100-hours），对于资源受限团队可能门槛较高。
